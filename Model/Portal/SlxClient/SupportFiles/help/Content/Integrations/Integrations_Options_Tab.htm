Integrations Options Tab

Use the Options tab to view and edit options that are specific to an integration. These options determine what and how information is synchronized between the integration and Infor CRM.

Integration options are represented by name-value pairs. Most integrations include a default set of name-value pairs that you can edit.

The Options tab is only available for integrations with a synchronization method of client-side sync or server sync.

How Do I?

Open the Options tab
  1. Open the Integration Detail view.
  2. Click the Options tab.
Edit options
  1. Click in the Value column for the option you want to edit.
    Use the Data Type column to identify the type of information to type as a value. For example, for a True/False data type, either "True" or "False" is a valid value.
  2. Type the new value.
  3. Click Save.

Set ION Options - Infor CRM Back Office Extension only. A Back Office Extension is an ERP system that is configured to send information to display in Infor CRM. Back Office Extension is managed by an administrator.

  1. Select the Tenant row, click in the Value column. Type the Tenant name that was defined when configuring the ERP connection in ION Desktop.
  2. Select the Logical ID row, click in the Value column. Type the Logical ID for your ERP. The Logical ID generated when configuring connection information for your ERP in ION Desktop.
  3. Infor CRM recommends leaving the remaining items as the default values, but they may be edited:
    • Success Message Expiration - The number of minutes after which success messages are purged from the log. The default setting is -1 which means that success messages are never purged from the log.
    • Error Message Expiration -The number of minutes after which error messages are purged from the log. The default setting is -1 which means that error messages are never purged from the log.
    • Log Success Messages - The default value is True, which means that the log will include successful transactions. To exclude successful transactions from the log, click in the Value column and type False.
    • Inbound Process List Size - The number of BODs to be processed by a single job service thread. The default value is 100.
    • Inbox Message Fetch Size - The maximum number of BODs that will be processed at one time. The default value is 1000.
    • Out of Sequence Message Expiration - Defines the length of time missing relationship information is stored in a log file. When BODs arrive out of sequence and they have missing dependent data, the system stores the missing relationships in a log.The recommended value is 720 hours.
  4. Click Save.
