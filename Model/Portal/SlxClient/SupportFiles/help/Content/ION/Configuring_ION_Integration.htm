Configuring ION Integration

Complete the following tasks to configure the ION Back Office integration.

The system administrator and users with the appropriate role can access these features.

To configure

  1. Run the SLXConversionUtility.exe.
    This needs to be done only once to support all integrations.
  2. Sign in as the administrator.
  3. On the Integrations List view, select ION Integration.
  4. Verify that the Enabled check box is selected.
    To turn off ION synchronization, clear the Enabled check box.
  5. Click Save.
  6. Click the Options tab and do the following:
    1. Select the Tenant row, click in the Value column. Type the Tenant name that was defined when configuring the ERP connection in ION Desktop.
    2. Select the Logical ID row, click in the Value column. Type the Logical ID for your ERP. The Logical ID generated when configuring connection information for your ERP in ION Desktop.
    3. Infor CRM recommends leaving the remaining items as the default values, but they may be edited:
      • Success Message Expiration - The number of minutes after which success messages are purged from the log. The default setting is -1 which means that success messages are never purged from the log.
      • Error Message Expiration -The number of minutes after which error messages are purged from the log. The default setting is -1 which means that error messages are never purged from the log.
      • Log Success Messages - The default value is True, which means that the log will include successful transactions. To exclude successful transactions from the log, click in the Value column and type False.
      • Inbound Process List Size - The number of BODs to be processed by a single job service thread. The default value is 100.
      • Inbox Message Fetch Size - The maximum number of BODs that will be processed at one time. The default value is 1000.
      • Out of Sequence Message Expiration - Defines the length of time missing relationship information is stored in a log file. When BODs arrive out of sequence and they have missing dependent data, the system stores the missing relationships in a log.The recommended value is 720 hours.
    4. Click Save.
  7. Click the Back Offices tab and verify that the following back offices are listed:
    • Infor Distribution A+
    • Infor Distribution SX.e
  8. On the Back Offices tab, select the desired back office link and use the Back Office Detail view to:

    		If the desired back office does not exist, you must first add a new back office.
    1. Configure the back office information boxes.
    2. Use the Accounting Entities tab to add a new accounting entity.
  9. (Optional) Use the ION BOD Mapping Detail view to:

    		If the desired ION BOD mapping does not exist, you must first add a new ION BOD mapping.
    1. Edit an existing ION BOD mapping.
    2. Add a BOD field mapping
    3. Edit a BOD field mapping
  10. Depending upon your ERP, you may need to use the Field Mapping Detail view to change the Account Type default value.
  11. Grant the appropriate access.
    1. To give individual users access to administrative integration views, add those users to the Integration role. These views include:
      • Back Office
      • ION BOD Mappings
      • ION Field Mappings
      • ERP Persons
    2. To give individual users access to the following non-administrative integration views, add those users to the BackOffice role. These views include:
      • Invoices
      • Receivables
      • Shipments
      • Bill To
      • Ship To
      • Pay From
      • ERP Persons (View)
