Defaulting the Account Type to Customer (Back Office Extension)

Infor CRM Back Office Extension only. A Back Office Extension is an ERP system that is configured to send information to display in Infor CRM. Back Office Extension is managed by an administrator. Some organizations may need to default the account type to Customer. Please refer to your ERP-specific integration guide to determine if this step is necessary.

In order to set a default account type, you must add a new field mapping to the Account BOD Mapping.

The system administrator and users with the appropriate role can access these features.

To add an ION BOD mapping

  1. On the Integrations List view, select ION Integration.
  2. Click the Back Offices tab, and select the appropriate back office.
  3. On the selected Back Office Detail view, click the BOD Mappings tab, and in the BOD Mapping column, click Account.
  4. On the Field Mappings tab, select a field mapping to clone.
    Select a mapping that has similar information or options. For example, ERP Status.
  5. On the ION Field Mappings Detail view, click the Save and New button.
    The Insert BOD Field Mapping view opens with some of the boxes already completed with information from the cloned field mapping.
  6. Make the following changes:
    1. In the Field Mapping box, type a unique and descriptive name for the field mapping.
    2. In the Default Value box, type Customer.
    3. Select Active to turn on synchronization.
    4. In the Property box type Type.
    5. In the Property XPath box change the first part of the path to Status1.
      For example, Status[@languageID='en-US' or @languageID='en-us' or @languageID = '' or not(@languageID)]
  7. Leave the remaining boxes with the default values.
  8. Click Save.
