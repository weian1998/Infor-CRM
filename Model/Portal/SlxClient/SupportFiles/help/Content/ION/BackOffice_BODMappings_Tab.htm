BOD Mappings Tab

Infor CRM Back Office Extension only. A Back Office Extension is an ERP system that is configured to send information to display in Infor CRM. Back Office Extension is managed by an administrator. Use the BOD Mappings tab to view and manage the BOD mappings that transfer information between Infor CRM and a Back Office extension.

The system administrator and users with the appropriate role can access these features.

How Do I?

Open the BOD Mappings tab
  • On the Back Office Detail view, click the BOD Mappings tab.
View BOD Mapping details
  • Click the BOD Mapping to open the ION BOD Mapping Detail view.
Add a field mapping
  1. Click the Insert New BOD Mapping button.
  2. Follow the steps for adding a BOD mapping.
Edit BOD mappings
  1. Select the BOD mapping you want to edit.
  2. Click in the grid to edit one or more of the following:
    • Noun - a description of the type of record to be mapped. If the same noun is used for multiple BOD mappings, only one can be active. Any other BOD mappings with same noun will have the Active check box cleared when another mapping with the same noun is added or designated "Active".
    • Entity - The Infor CRM entity, or record type, being mapped. Type the name of an Infor CRM entity type. For example, "Account".
    • Priority - Specifies the order in which BODs are processed.Type any 3-digit number to designate the BOD Priority. The BOD with the lowest number will be processed first and all other BODs will be processed in ascending order.
    • Handler Class - Use this box to enter a custom handler. A custom handler is a fully qualified name of a user-defined class that extends the class for the particular URL protocol.
    • Active - Use this option to turn on or turn off synchronization for this BOD mapping.
      To undo any changes, click Reset before clicking the Save button.
  3. Click Save.
