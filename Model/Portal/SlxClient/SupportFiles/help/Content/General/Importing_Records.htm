Importing Records

Use the Import view to import lists of contacts into Infor CRM. You may only import records from a comma-separated-values (CSV) list. Valid file types include Microsoft Excel, Microsoft Outlook export files, .txt files, and other database file types, but files must be saved as a CSV file.

The system administrator and users with the appropriate role can access these features.

To import

  1. Review the file you want to import and verify that it contains all of the data you want to import into Infor CRM. Example

    The import list must contain a column that is mapped to the Infor CRM Account.Account Name field and each record must contain data in that column. Any records without this inofrmation will not be imported.

    • If the file does not contain a column for that information, edit the import file to include a column and data for each record to be imported.
    • If a record does not include data in the column that is mapped to the Infor CRM Account.Account Name field, update the import file to include data in the column.

    Example: In order for address information to be imported for both a contact and an account, the address information must be contained in separate columns that can be mapped to either the contact or the account. If the contact and account information is the same, then copy and paste each address column and give them distinct names so that they can be mapped separately in step 11.

  2. Open the Contacts List or Contact Detail view.
  3. From the Task Pane, under Common Tasks, select Import.
  4. Click Browse, browse to the list you want to import, and then click Open.
    The Add Attachment(s) for dialog box opens.
  5. In the Description box, enter a description for the attachment if the file name is not clear.
  6. Click OK.
    The selected file appears next to the Browse button.
  7. If you want to use a saved import template, click the Import Template drop-down arrow and select from the list.

    A template defines import settings such as mappings, import type, matching criteria, default owner, and ad hoc group settings.

  8. Click Next.

  9. On the Define Delimiter view, read the text at the top of the page, follow any recommendations, and review the records preview.

    • If the preview looks correct, then click Next to continue the import.
    • If necessary, change the options until the records preview looks correct.
      1. If you need to change the delimiter, select one of the following options:
        • Tab
        • Semicolon
        • Comma
        • Space
        • Other
      2. If you selected Other, you must specify the delimiter used to separate fields in the list. Type the delimiter character in the Other box.
      3. From the Text Qualifier list, click the drop-down arrow and select a qualifier from the list.
        A text qualifier identifies text that should be imported exactly as it appears.
      4. If the list contains field names, select the First Row Contains Field Names check box to display the field names in the Preview area.
        If there are extended characters in the imported file that do not correctly display in the Preview area, you must resave the import file and explicitly define the encoding type as UTF-8, Unicode, or ANSI. After you resave the file, click the Back button to return to the Select a File step and reselect the updated file before continuing the import.
      5. Click Next
  10. Review or edit the import options.
    1. Define the matching criteria to be used for identifying duplicate records in Infor CRM. Example
      The options available depend on the type of record being imported.

      Example: If importing contacts, matching Contacts based on Email will flag any records with the same email address as duplicate records.

      • Click the drop-down arrow and select from the list for each available match criteria.

      It is important to select match criteria that is most likely to be unique to a record. For example, an Email address is more likely to be unique to a record than Last Name, since many people may share a common last name.

    2. Click the Default Owner Find button to select the person or team who will own the imported records.
      The logged in user will be assigned as the account manager.

    3. Set ad hoc group options.

      • To add the imported records to a new group, select Create a new ad hoc group. Note the name of the group to be created.
      • To add the imported records to an existing group, select Add to existing ad hoc group, and then select a group from the list.

    4. Click Next.
  11. The import will automatically map as many fields as possible. If necessary, use the Map Fields view to map any remaining fields.

    Some fields are required by Infor CRM and must be mapped in order for the import to complete successfully.

    1. Click View Unmapped Fields to see only the fields that are not mapped.
      • For each field listed in the Import Field column, click the drop-down arrow in the Infor CRM Field column and select the Infor CRM field that you want to map to the import field.
        As fields are mapped, they will be removed from the View Unmapped Fields list and the count will decrease.
    2. (Optional) Click View Mapped Fields to see only fields that are mapped.
    3. (Optional) Click View All Fields to review all of the fields in the import file and review the mappings to Infor CRM fields.
  12. Click Next.
  13. In the Import Detail Preview, review your import settings.
  14. If you want to save your import settings as a template to be used for future imports, do one of the following:
    • If you want to update an existing import template, click the Import Template drop-down arrow, select the template you want to update, and then click Update Current Template.
      The existing template will be saved with any import setting changes.
    • If you want to create a new import template, click Save As Import Template, in the Description box, type a brief description of the import template, and then click Save.
      The new import template description is added to the Import Template list on the Import Leads dialog box.
  15. Click Import Now.

Next Steps

View imported records
  • Do one of the following:
    • Open the Job Notifications alert in the menu bar, if the import completed successfully and all records were imported, then click the link to open the ad hoc group created in step 10c.
    • Open the Contacts List view and click the tab for the ad hoc group created in step 10c.
View import history record
  • Do one of the following:
    • Open the Job Notifications alert in the menu bar, if the import completed but not all records were imported, then click the link to open the Import History detail view.
    • On the Tools menu, click Import History to open the Import History List view, and click the Import Number of the import history record you want to view.
