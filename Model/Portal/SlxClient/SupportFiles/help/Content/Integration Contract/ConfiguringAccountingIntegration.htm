Configuring X3 Accounting Integration

Complete the following tasks to configure an X3 accounting integration.

  1. If multi-currency is enabled, ensure Infor CRM and your accounting system have the same base currency.
  2. Run the SLXConversionUtility.exe.
    This needs to be done only once to support all integrations.

    Windows Authentication must be configured for the administrative user before running the conversion utility. For detailed steps, see the "Using Windows Authentication" topic in the Administrator Help

  3. Configure Infor CRM SData synchronization.
    This needs to be done only once to support all integrations.
  4. On the Integrations List view, select X3.
  5. Review the information boxes.
  6. In the Link Type, make sure System is selected
  7. Select the Enabled check box.
  8. Set the synchronization options.
    1. In the Sync Every box, type the number of minutes that should pass between the beginning of synchronization cycles.
    2. In the Run sync between boxes, click the Calendar button and select the beginning time and end time during which synchronization should occur. Infor CRM recommends that selected times not cross midnight.
  9. Click Save
  10. On the Links tab, click the Add Link button.
    1. In the URL, type the SData URL that Infor CRM uses to share information between systems.
      • Use one of the following formats:
        • If your implementation does not use the default port (port 80), use format http://server:port/sdatavirtualdir/application/contract/dataset/
        • If your implementation uses port 80, omit the port number from the URL.
          The Sync Service automatically strips the default port from the URL which causes matching issues and errors. Avoid this by using format http://server/sdatavirtualdir/application/contract/dataset/
      • Substitute values provided by your integration partner:
        •  Sdatavirtualdir - For example, sdata.

        •  application - your accounting application name. For example, accounts50.

        •  contract - the contract name. For example, GCRM.
        •  dataset - distinguishes a data set or operating company. For example, /-/

    2. In the Display Name box, type a brief, but descriptive name.
    3. In the User Name box, type the user name for the URL.
    4. In the Password, type the password for the URL.

    5. Click the Test Link button and view the information in the Status box.
      If the feed did not test successfully, review the link information, make any changes, and test the link again.

    6. Click OK.
  11. Compare the database schema in Infor CRM with your integration schema to ensure data integrity.

    If you have customized Infor CRM or if you created custom entities in the integration that do not match Infor CRM, synchronization of that information may not work.

  12. Use the Options tab to set X3-specific options.
    1. Next to each option, click in the Value column and type True to enable or False to disable the following options
      • AccountingSystemManagesSO - If True, removes the ability to create and edit sales orders in Infor CRM. The accounting system must be used to manage sales orders.
      • EnableEmailAuthentication - If True, then email authentication is used to restrict data that can be seen or to audit what a user has viewed. Some accounting systems may require email authentication. After enabling this option, you must modify each user's profile to include an email address that matches a user's email address in the accounting system.
      • RestrictToSingleAccountingSystem - If True, limits your integration to Infor CRM and one accounting system.
    2. Click Save
  13. On the SData Sync Configuration view, click the Matching tab.
    1. Click Edit next to a resource to set global default matching criteria for users creating links.
    2. Select the Property and Operator for the match criteria.
    3. To add additional matching criteria, click .
    4. When finished, click OK.
  14. Repeat step 13 for each resource as necessary.
  15. Configure the Integration Synchronization Service. This needs to be done only once to support all integrations.
  16. (If necessary), export and merge records from your accounting system.
  17. Configure the user roles for this integration.
    • The Integration Contract Lock Price role grants permission to lock product pricing on a sales order.

    • The PromoteAccount action grants users permission to promote accounts and is part of the Standard User role by default.
