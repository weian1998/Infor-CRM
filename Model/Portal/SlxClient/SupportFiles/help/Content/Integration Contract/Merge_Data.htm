Merge Data

(Accounting Integration Only) The Merge Data screen displays differences in data when you are linking records between Infor CRM and an accounting system. Use this screen to select the data you want to use when merging records. The data you keep overwrites the data in the other system. Tell me more...

  • If Accounting Integration is not enabled and you merge two accounts, sales orders associated to the deleted account are added to the merged account.
  • If Accounting Integration is enabled and you merge two accounts, an account that has associated sales orders or quotes that have been submitted to the accounting system cannot be deleted (the account that contains the submitted records cannot be the Source account). If necessary, you can control the Source account by selecting that account first when merging data. This allows you to keep the account with associated sales orders by merging into it.

To select

  1. Open the Linking wizard and proceed through the previous screens.
  1. Complete the Merge Data screen.
    BoxDescription/Action
    Property

    Lists the data fields that exist for the record in the two records in Infor CRM.

    [Source] column

    Use the column to the left of the Source column to select, on a field-by-field basis, which data to keep from the Infor CRM account (Source) when the records are merged.

    Select the option in the column heading to keep all the Infor CRM (Source) data.

    [Target] column

    Use the column to the left of the Target column to define, on a field-by-field basis, which data to keep from the accounting system (Target) when the records are merged.

    Select the option in the column heading to keep all accounting system data.

    Hide Duplicates or Show AllClick to show or hide the property information that is the same in both systems. By default, only fields with different values display in the list.
  2. Click Next to go to the next step.
    Depending on your choices, you may need to merge contacts .
