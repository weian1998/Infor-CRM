Grundlegendes zu Benutzern

Benutzer sind die Personen, die auf Ihre Infor CRM-Datenbank zugreifen dürfen. Jeder Benutzer hat ein Profil, in dem z. B. folgende Informationen festgelegt werden: Benutzername, Titel, Manager, Teammitgliedschaft, Abteilungsmitgliedschaft und Sicherheitszugriff.

Sie können die folgenden Benutzertypen hinzufügen:

  • Netzwerk

Ein Benutzer, der hauptsächlich über eine direkte Netzwerkverbindung mit der Datenbank auf Infor CRM zugreift.

  • Gleichzeitig

Diese Lizenzen ermöglichen einen Zugriff auf Infor CRM, sind aber nicht einzelnen Benutzern zugeordnet. Die Anzahl der Benutzer, die zur gleichen Zeit auf die Software zugreifen, darf nicht über die Anzahl der gewährten Lizenzen für gleichzeitige Benutzer hinausgehen. Wenn Sie beispielsweise 100 Lizenzen für gleichzeitige Benutzer haben, können Sie eine beliebige Anzahl gleichzeitiger Benutzer erstellen. Allerdings können nur 100 dieser Benutzer zum gleichen Zeitpunkt bei Infor CRM angemeldet sein.

Gleichzeitige Benutzer können sich bei Infor CRM Web Client, Infor CRM Client, Administrator, Architect, SpeedSearch-Client, Sync-Client (nur Remote Office) und Remote Office anmelden. Bei jeder Anmeldung wird eine Lizenz für gleichzeitige Benutzer in Anspruch genommen. Gleichzeitige Benutzer, die sich beim SpeedSearch-Client anmelden, nehmen unabhängig von der Zugriffsmethode keine Lizenz für gleichzeitige Benutzer in Anspruch.

Lizenzen für gleichzeitige Benutzer können zwischen dem Host und einem bzw. allen Remote Offices aufgeteilt werden. Diese Zuweisung kann jederzeit in Administrator (Systeme > Büros > Remote Office bearbeiten > Registerkarte Sync-Optionen) geändert werden. Wenn Sie einem Remote Office Lizenzen für gleichzeitige Benutzer zuweisen, geht der Host davon aus, dass zu jedem Zeitpunkt alle Remote Office-Lizenzen in Anspruch genommen werden. Wenn Sie z. B. über eine Lizenz für 30 gleichzeitige Benutzer verfügen und einem Remote Office 10 dieser Lizenzen zuweisen, zieht der Host die 10 Lizenzen automatisch ab und lässt nur noch 20 Benutzer in der Hauptniederlassung gleichzeitig auf die Anwendungen zugreifen.

Der Lizenztyp "Gleichzeitiger Benutzer" kann Remote-Benutzern nicht zugewiesen werden.

  • Remote

Remote-Clients haben einen Teilbestand der Hauptstandortdatenbank auf ihrem Computer (z.B. auf einem Laptop) und gleichen mit Hilfe der Synchronisierung Änderungen zwischen ihrer Datenbank und der am Hauptstandort ab. Beachten Sie, dass Remote-Benutzer niemals eine Verbindung mit dem Remote-Office, sondern immer mit der Hauptniederlassung herstellen.

Wenn Ihre Einrichtung weniger als fünf Remote-Benutzer hat und es nicht wichtig ist, ob diese Benutzer gemeinsamen oder unmittelbaren Zugriff auf dieselben Firmen haben, kann jeder Benutzer als Remote-Benutzer konfiguriert werden. Beispiele für Remote-Benutzer sind Vertriebsmitarbeiter im Außendienst oder einzelne Regionalvertreter an abgelegenen Standorten.

  • Vorlage

Benutzervorlagen können erstellt werden, um Einstellungen zu definieren, die Sie bei der Erstellung neuer Benutzer verwenden möchten. Sie können beispielsweise eine Benutzervorlage erstellen, die alle Optionen für gleichzeitige Benutzer enthält. Wenn Sie dann einen neuen gleichzeitigen Benutzer erstellen, können Sie die Vorlage für gleichzeitige Benutzer mit vordefinierten Profileinstellungen verwenden.

  • Benutzervorlagen haben weder Einfluss auf die Lizenzanzahl, noch werden sie in der Benutzerliste angezeigt.
  • Eine Benutzervorlage ist kein aktiver Benutzer und kann sich nicht bei Infor CRM anmelden.
  • Nur Web

Benutzer, die nur über Zugriff auf den Infor CRM Web Client verfügen und die Informationen ihren Zugriffsrechten entsprechend anzeigen, hinzufügen, löschen oder bearbeiten können. Dieser Benutzertyp kann sich nicht beim Infor CRM Client anmelden.

  • Add-on-Benutzer

Mithilfe des Add-on-Benutzers können Sie für Client-Schnittstellen von Drittanbietern einen Benutzer mit Zugriff auf den Infor CRM OLE DB Provider erstellen. Dieser Benutzer kann sich bei keiner der Infor CRM-Anwendungen anmelden.

  • WebViewer

WebViewer-Benutzer haben schreibgeschützten Zugriff auf Firmen, Kontakte, Verkaufschancen und Tickets. Je nach Firmen- und Kalendersicherheit können WebViewer-Benutzer Aktivitäten, Notizen und Historien hinzufügen, bearbeiten und löschen, jedoch keine Prozesse verwenden. Dieser Benutzertyp kann sich nicht beim Infor CRM Client anmelden.

Ein WebViewer-Benutzer kann Folgendes weder sein noch tun:

  • Vorgesetzter
  • Zugriffsberechtigter für ein Team
  • Andere Benutzer auf der Registerkarte "Benutzerteammitglieder" anzeigen

Darüber hinaus unterstützt Infor CRM mehrere besondere Benutzertypen:

  • Systemadministrator

Ein Benutzer mit Zugriff auf alle Funktionen von Infor CRM. Der Systemadministrator meldet sich als "admin" an.

  • Verwaltung

Ein Benutzer, der Aufgaben durchführen kann, die normalerweise dem Systemadministrator vorbehalten sind. Benutzer mit Zugriff auf Verwaltungsfunktionen werden mithilfe administrativer Rollen in Administrator oder von Rollen im Web Client definiert. Verwaltungsbenutzer melden sich mit ihrem Benutzernamen an (z. B. Leo oder Daniel). Der Systemadministrator bleibt der einzige Benutzer, der über vollständigen Zugriff auf alle Funktionen im System verfügt.

  • Inaktiv

Dieser Benutzertyp bietet Ihnen eine Möglichkeit, die Aktivitätshistorie von Benutzern beizubehalten, die aus Ihrem Unternehmen ausscheiden. Ausgeschiedene bzw. entfernte Benutzer werden insofern wie deaktivierte Benutzer behandelt, als ihre Kontaktaktivitäten auch dann in der Datenbank gespeichert bleiben, wenn sie selbst nicht mehr auf das System zugreifen können. Allerdings verwenden deaktivierte Benutzer weiterhin Lizenzen, während ausgeschiedene Benutzer keine Lizenzen mehr in Anspruch nehmen.
