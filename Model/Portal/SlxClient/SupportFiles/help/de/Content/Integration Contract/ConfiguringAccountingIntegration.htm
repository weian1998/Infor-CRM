Konfigurieren der X3-Buchhaltungsintegration

Führen Sie die folgenden Aufgaben aus, um eine X3-Buchhaltungsintegration zu konfigurieren.

  1. Wenn mehrere Währungen aktiviert sind, muss sichergestellt werden, dass Infor CRM und Ihr Buchhaltungssystem dieselbe Basiswährung verwenden.
  2. Führen Sie die Datei SLXConversionUtility.exe aus.
    Dies muss nur einmal durchgeführt werden, um alle Integrationen zu unterstützen. 

    Für den Benutzer mit Administratorrechten muss vor der Ausführung des Konvertierungsdienstprogramms die Windows-Authentifizierung konfiguriert werden. Einzelheiten dazu finden Sie unter dem Thema "Verwenden der Windows-Authentifizierung" in der Administrator-Hilfe.

  3. Konfigurieren Sie die Infor CRM SData-Synchronisierung.
    Dies muss nur einmal durchgeführt werden, um alle Integrationen zu unterstützen.
  4. Wählen Sie in der Listenansicht "Integrationen" die Option X3.
  5. Überprüfen Sie die Informationsfelder.
  6. Im Feld Verknüpfungstyp muss System markiert sein.
  7. Markieren Sie das Kontrollkästchen Aktiviert.
  8. Legen Sie die Synchronisierungsoptionen fest.
    1. Geben Sie im Feld Synchronisieren alle die Anzahl der Minuten ein, die zwischen den Anfängen der Synchronisierungszyklen jeweils vergehen sollen.
    2. Klicken Sie in den Feldern Synchronisierung ausführen zwischen auf die Schaltfläche  Kalender  , und wählen Sie die Anfangs- und Endzeiten aus, zwischen denen die Synchronisierung stattfinden soll. Infor CRM empfiehlt, keine Zeiten über Mitternacht auszuwählen.
  9.  Klicken Sie auf Speichern
  10. Klicken Sie auf der Registerkarte Verknüpfungen auf die Schaltfläche Verknüpfung hinzufügen.
    1. Geben Sie im URL-Feld den SData-URL ein, der Infor CRM für die Freigabe von Informationen zwischen den Systemen verwendet wird.
      • Verwenden Sie eines der folgenden Formate:
        • Falls Ihre Implementierung nicht den standardmäßigen Port (Port 80) nutzt, verwenden Sie das folgende Format http://server:port/sdatavirtualdir/Anwendung/Vertrag/Datensatz/
        • Verwendet Ihre Implementierung jedoch Port 80, geben Sie die Port-Nummer nicht im URL an.
          Der Synchronisierungsdienst entfernt den Standard-Port aus dem URL, was zu Zuordnungsproblemen und Fehlern führt. Sie vermeiden dies durch Verwendung des folgenden Formats: http://server/sdatavirtualdir/application/contract/dataset/
      • Von Ihrem Integrationspartner bereitgestellte Ersatzwerte:
        •  Sdatavirtualdir - Beispiel: sdata.

        •  Anwendung - der Name Ihrer Buchhaltungsanwendung. Beispiel: Firma50.

        •  Vertrag - Der Name des Vertrags. Beispiel: GCRM.
        •  Datensatz - unterscheidet einen Datensatz oder ein aktives Unternehmen. Beispiel: /-/

    2. Geben Sie im Feld Anzeigename einen kurzen, aber eindeutig beschreibenden Namen ein.
    3. Geben Sie im Feld Benutzername den Benutzernamen für den URL ein.
    4. Geben Sie im Feld Passwort das Passwort für den URL ein.

    5. Klicken Sie auf die Schaltfläche Link testen, und zeigen Sie die Informationen im Feld Status an.
      Wenn der Feed nicht erfolgreich getestet wurde, prüfen Sie die Link-Informationen, nehmen Sie die erforderlichen Änderungen vor, und testen Sie den Link erneut.

    6. Klicken Sie auf OK.
  11. Vergleichen Sie das Datenbankschema in Infor CRM mit dem Ihrer Buchhaltungsanwendung, um die Datenintegrität zu gewährleisten.

    Wenn Sie Infor CRM benutzerspezifisch angepasst oder benutzerspezifische Entitäten in der Integration erstellt haben, die nicht mit Infor CRM übereinstimmen, werden diese Informationen evtl. nicht synchronisiert.

  12. Verwenden Sie die Registerkarte Optionen, um X3-spezifische Optionen festzulegen.
    1. Klicken Sie neben jeder Option in die Spalte "Wert", und geben Sie "Wahr" oder "Falsch" ein, um die folgenden Optionen zu aktivieren bzw. deaktivieren
      • AccountingSystemManagesSO – Falls der Wert für diese Option "Wahr" ist, können keine Verkaufsaufträge in Infor CRM erstellt und bearbeitet werden. Zur Verwaltung von Verkaufsaufträgen muss das Buchhaltungssystem verwendet werden.
      • EnableEmailAuthentication - Falls der Wert für diese Option "Wahr" ist, wird die E-Mail-Authentifizierung verwendet, um anzeigbare Daten zu beschränken oder um zu überprüfen, was ein Benutzer angezeigt hat. Bei manchen Buchhaltungssystemen ist die E-Mail-Authentifizierung erforderlich. Nach der Aktivierung dieser Option müssen Sie die einzelnen Benutzerprofile ändern, sodass diese eine E-Mail-Adresse enthalten, die mit der E-Mail-Adresse eines Benutzers im Buchhaltungssystem übereinstimmt.
      • RestrictToSingleAccountingSystem -– Wenn der Wert für diese Option "Wahr" ist, wird die Integration auf Infor CRM und ein Buchhaltungssystem beschränkt.
    2. Klicken Sie auf Speichern 
  13. Klicken Sie in der SData Sync-Konfigurationsansicht auf die Registerkarte Abgleich.
    1. Klicken Sie auf Bearbeiten neben einer Ressource, um ein globales, standardmäßiges Übereinstimmungskriterium für Benutzer festzulegen, die Verknüpfungen erstellen.
    2. Wählen Sie Eigenschaft und Operator als Übereinstimmungskriterien.
    3. Klicken Sie zum Hinzufügen zusätzlicher Übereinstimmungskriterien auf .
    4. Klicken Sie auf OK, wenn Sie fertig sind.
  14. Wiederholen Sie ggf. Schritt 13 für jede Ressource.
  15. Konfigurieren Sie den Synchronisierungsdienst für die Integration. Dies muss nur einmal durchgeführt werden, um alle Integrationen zu unterstützen.
  16. Bei Bedarf können Sie Datensätze aus dem Buchhaltungssystem exportieren und zusammenführen.
  17. Konfigurieren Sie die Benutzerrollen für diese Integration.
    • Die Rolle "Integration Contract - Preis sperren" berechtigt zum Sperren von Produktpreisen auf einem Verkaufsauftrag.

    • Die Aktion "Firma hochstufen" gibt Benutzern die Berechtigung, Firmen hoch zu stufen und gehört standardmäßig zur Rolle "Standardbenutzer".
