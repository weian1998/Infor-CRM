Kaskadierungsoptionen

Select in
Löschen

Löscht den ausgewählten Datensatz sowie alle abhängigen Datensätze.

Wenn beispielsweise ein Benutzer aus der Tabelle BENUTZERINFO gelöscht wird, werden die verknüpften Datensätze in der Tabelle BENUTZERPROFIL ebenfalls gelöscht.
Löschen

Das Schlüsselfeld, das den Datensatz in der sekundären Tabelle verknüpft, wird gelöscht, der gesamte Datensatz in der sekundären Tabelle bleibt jedoch erhalten.

Wenn Sie zum Beispiel den Datensatz KONTAKT löschen, werden die Datensätze in HISTORIE, die mit diesem Kontakt verknüpft sind, nicht gelöscht. Es wird nur der Join mit diesem bestimmten Kontakt gelöscht.
Ersetzen

Beim Löschen eines Datensatzes wird das Schlüsselfeld, durch das der Datensatz in der sekundären Tabelle verknüpft ist, gelöscht, aber nicht der Datensatz selbst. Ersetzen Sie das Schlüsselfeld durch eine andere ID.

Wenn beispielsweise ein Benutzer aus der Tabelle BENUTZERINFO gelöscht wird, können die Firmen dieses Benutzers mit der Option "Ersetzen" anderen Benutzern zugewiesen werden.
Löschen (Kaskadierung stoppen)

Löscht den ausgewählten Datensatz, jedoch nicht die abhängigen Datensätze.

Wenn beispielsweise ein Benutzer aus der Tabelle BENUTZERINFO gelöscht wird, ändern sich die verknüpften Datensätze in HISTORIE nicht.
Ignorieren (keine Kaskadierung) Wenn Sie einen Datensatz löschen, überspringen Sie diese Tabelle.
