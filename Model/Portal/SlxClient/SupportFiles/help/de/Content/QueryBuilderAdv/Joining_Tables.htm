Erstellen von Tabellen-Joins

Die Verbindung zwischen zwei Tabellen wird als Join bezeichnet. Ein Join wird mithilfe eines Felds eingerichtet, das in beiden Tabellen vorhanden ist. Dieses Feld kann in den beiden Tabellen unterschiedlich heißen, muss aber dieselben Informationen enthalten. Es gibt zwei Arten von Joins: Local Joins und Global Joins.

Local Joins

Ein Local Join ist temporär. Ein Local Join existiert nur dann, wenn das betreffende Objekt oder die betreffende Abfrage aktiv ist bzw. ausgeführt wird. Local Joins werden in der Verzeichnisstruktur des Abfrage-Assistenten nicht angezeigt.

So öffnen Sie das Dialogfeld "Local Join"

  • Klicken Sie im Abfrage-Assistenten auf ein Feld und anschließend auf Local Join erstellen.

Global Joins

Ein Global Join wird in der Verzeichnisstruktur des Abfrage-Assistenten von einer Session zur nächsten beibehalten. Infor CRM enthält vordefinierte Global Joins (System-Joins), die nicht gelöscht werden können. Die vordefinierten Global Joins können von allen Benutzern verwendet werden, aber nur der Systemadministrator oder Benutzer mit Administratorrechten können diese mithilfe des Global Join-Managers erstellen.

So öffnen Sie den Global Join-Manager

  • Klicken Sie im Abfrage-Assistenten auf die Schaltfläche Joins (während Sie als Administrator angemeldet sind).

Arbeiten mit Joins

Beim Erstellen eines Joins müssen Sie eine übergeordnete und eine untergeordnete Tabelle angeben. Der Typ des Joins legt fest, welche Daten von der Abfrage zurückgegeben werden. Es gibt drei Arten von Joins: Inner, Left und Right Join.

Beispielsweise enthalten die Tabellen KONTAKT und FIRMA beide das Feld "Firmen-ID". Dementsprechend können Sie das Feld "Firmen-ID" zum Verknüpfen der Tabellen FIRMA und KONTAKT verwenden.

Tabellen können auch mithilfe von Feldern verknüpft werden, die über unterschiedliche Namen, aber identische Werte verfügen. Beispielsweise ist die Tabelle FIRMA mit der Tabelle BENUTZERINFO über das Feld "Account Manager-ID" in der Tabelle FIRMA und das Feld "Benutzer-ID" in der Tabelle BENUTZERINFO verknüpft. Obwohl der Feldname in den Tabellen unterschiedlich ist, sind die Werte in den Feldern identisch.

Grundsätzlich werden Joins zum Erstellen von Abfragen verwendet, die anhand einer Datenbank ausgeführt werden. Sobald ein Join erstellt ist, kann jedes Feld in jeder Tabelle in einer Abfrage verwendet werden. Diese Abfragen werden mithilfe des Abfrage-Assistenten erstellt.
