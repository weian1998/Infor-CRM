ID-Felder in Tabellen

Jede Tabelle weist mindestens ein ID-Feld auf, das verwendet wird, um die Eindeutigkeit eines jeden Datensatzes in der Tabelle sicherzustellen. Wenn ein Datensatz zur Tabelle FIRMA hinzugefügt wird, wird ein Wert automatisch in das Feld FIRMENID eingegeben. Jedes Mal, wenn ein Datensatz hinzugefügt wird, wird ein Wert basierend auf den Informationen in der Schlüsseldatei erzeugt.

Welche Felder die wichtigen ID-Felder einer Tabelle sind, ist von der Art der Beziehung zwischen dieser und anderen Tabellen abhängig. Jede erstellte Tabelle, die nicht mit einer anderen Tabelle verknüpft ist, verfügt über ein eigenes, eindeutiges ID-Feld. Wenn eine neue Tabelle als untergeordnete Tabelle mit einer 1:1-Beziehung zu einer anderen Tabelle erstellt wird, ist deren ID-Feld gleich dem ID-Feld der übergeordneten Tabelle. Wenn die untergeordnete Tabelle eine 1:n-Beziehung zur anderen Tabelle aufweist, ist mehr als ein Feld erforderlich, um Eindeutigkeit sicherzustellen. Deshalb werden beim Erstellen der Tabelle automatisch zwei IDs generiert: eine, die für die Tabelle eindeutig ist, und eine, die das ID-Feld der Tabelle darstellt, auf der sie basiert. Wenn es sich bei der Tabelle um eine Verknüpfungstabelle handelt, deren Zweck in der Auflösung einer n:n-Beziehung besteht, werden drei ID-Felder benötigt. Infor CRM erzeugt automatisch nur zwei IDs, nämlich eine, die für die Verknüpfungstabelle eindeutig ist, und eine für die Primärtabelle. Das ID-Feld der untergeordneten Tabelle müssen Sie bei der Erstellung der Tabelle hinzufügen, andernfalls werden über- und untergeordnete Tabellen nicht verknüpft.

Zwar beinhalten einige der in Infor CRM vorhandenen Verknüpfungstabellen nur zwei ID-Felder, Sie sollten jedoch bei allen selbst erstellten Verknüpfungstabellen mit drei ID-Feldern arbeiten.
