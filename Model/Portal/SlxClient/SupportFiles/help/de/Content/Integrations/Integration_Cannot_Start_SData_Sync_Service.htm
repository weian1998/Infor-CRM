Beheben von Integrationen - Infor CRM-SData-Synchronisierungsdienst lässt sich nicht starten

Beim Synchronisieren von Informationen zwischen Infor CRM und unterstützten Buchhaltungsanwendungen können folgende Probleme auftreten.

Problem:

Infor CRM-SData-Synchronisierungsdienst lässt sich nicht starten

Lösungen:

  • Stellen Sie sicher, dass die Windows-Authentifizierung aktiviert und dem Benutzer zugeordnet ist, in dessen Namen der Dienst angemeldet und ausgeführt werden soll.

    Infor CRM empfiehlt für diesen Benutzer eine Lizenz für externen Zugriff und keine ADMIN-Lizenz zu verwenden.

  • Überprüfen Sie, ob die Verbindung in der Datei SLXSDataSyncServer.exe.config richtig konfiguriert ist. Weitere Informationen...
    • Die Verbindungsinformationen müssen der für Ihre Website bereitgestellten Datei connection.config entsprechen.
      Standardmäßig wird die Datei SLXSDataSyncServer.exe.config in ...\Programme\Saleslogix\ oder ...\Programme (x86)\Saleslogix\ , und die Verbindungsdatei unter ...\inetpub\wwwroot\slxclient
    bereitgestellt.
  • Stellen Sie sicher, dass die Konfiguration der SData-Synchronisierung korrekt mit den richtigen Angaben für Infor CRM Feed, Benutzername und Passwort eingerichtet ist.
  • Stellen Sie sicher, dass die Integration(en) korrekt in der Integrationsdetailansicht eingerichtet ist/sind.
    • Stellen Sie sicher, dass Synchronisierung aktiviert ist.
  • Öffnen Sie SLXSDataSyncServer.exe im Eingabeaufforderungsfenster, um die Ausgabe der Anwendung anzuzeigen.
  • Sehen Sie sich protokollierte Fehler, Warnungen und Debug-Meldungen an. Weitere Informationen...
    • Öffnen Sie die Detailansicht "Sync-Aufträge", öffnen Sie die Registerkarte "Protokollelemente", und filtern Sie nach Protokolltyp, um Fehler, Warnungen oder Debug-Meldungen anzuzeigen.
  • Überprüfen Sie auf dem Web-Server die Windows-Ereignisanzeige auf Fehler.
  • Aktivieren Sie zusätzliche Protokollierung auf dem Web-Server. Weitere Informationen...
    1. Erstellen oder öffnen Sie im Infor CRMInstallationsordner von die Datei log4net.config, um Debug-Meldungen von SLXSDataSyncServer in der Ereignisanzeige auszugeben.
    2. Fügen Sie folgenden Eintrag hinzu:
      <log4net threshold="ALL">
      <appender name="EventLogAppender"
      type="log4net.Appender.EventLogAppender" >
      <applicationName value="Services" />
      <layout type="log4net.Layout.PatternLayout">
      <conversionPattern value="%date [%thread] %-5level %logger
      [%property{NDC}] - %message%newline" />
      </layout>
      </appender>

      <logger name="SDataSync">
      <level value="ALL" />
      <appender-ref ref="EventLogAppender"/>
      </logger>
      </log4net>
    3. Öffnen Sie den Ordner ...\Inetpub\wwwroot\SLXClient, öffnen Sie die Datei log4net.config und wiederholen Sie Schritt b. Dadurch können Debug- und Informationsmeldungen für SData-Prozesse ausgegeben werden, die vom Client initiiert wurden, z. B. Aufrufe des Preisfestsetzungsdiensts.
    4. Öffnen Sie im Ordner ...\Inetpub\wwwroot\sdata die Datei log4net.config, und ändern Sie den SDataSync-Eintrag level value von INFO in ALL oder DEBUG, je nachdem, welche Meldungen Sie auf der Registerkarte Protokollelemente anzeigen möchten.
      <logger name="SDataSync">
      <level value="INFO" />
      <appender-ref ref="EventLogAppender"/>
