Registerkarte "Aktionen"

Auf der Registerkarte "Aktionen" können Sie die einer Rolle zugewiesenen gesicherten Aktionen verwalten. Rollen bestimmen, auf welche Funktionalität Benutzer im Web Client zugreifen können. Durch Hinzufügen gesicherter Aktionen zu einer Rolle wird bestimmt, auf welche Funktionen die Rolle Zugriff hat. Durch Hinzufügen von Benutzern zur Rolle wird bestimmt, wer Zugriff hat.

Weitere Informationen...

Elemente werden in Application Architect abgesichert, indem ihnen eine Aktion zugewiesen wird oder Sicherheitseinstellungen angewendet werden. Nachdem eine abgesicherte Aktion eingerichtet wurde, können Benutzer nur dann auf das dazugehörige Element zugreifen, wenn ihnen eine Rolle zugewiesen wurde, die die abgesicherte Aktion enthält. Wählen Sie beispielsweise Entitäten/Firma/Speichern als Wert der Eigenschaft Angewandte Sicherheit für die Schaltflächenaktion Speichern in der Hauptansicht Firma. Wenn diese Schaltfläche zur Laufzeit geladen wird, ändert sich der Anzeigewert abhängig davon, ob der aktuelle Benutzer die Berechtigung zum Ausführen der Aktion hat.

Weitere Informationen finden Sie in der Application Architect-Hilfe im Thema "Grundlegendes zu gesicherten Aktionen".

Beim Hinzufügen oder Entfernen einer gesicherten Aktion für eine Rolle auf dieser Registerkarte wird die gesicherte Aktion auch in der Detailansicht "Gesicherte Aktion" auf der Registerkarte Rollen hinzugefügt oder entfernt.

Vorgehensweise

Hinzufügen einer gesicherten Aktion zu einer Rolle

  1. Öffnen Sie die Rollendetailansicht.
  2. Klicken Sie auf die Registerkarte Aktionen.
  3. Klicken Sie auf Aktion hinzufügen.
  4. Wählen Sie über die Funktion Suchen die Aktionen aus, die Sie der Rolle hinzufügen möchten.
  5. Klicken Sie auf Hinzufügen.

Entfernen einer gesicherten Aktion aus einer Rolle

  1. Öffnen Sie die Rollendetailansicht.
  2. Klicken Sie auf die Registerkarte Aktionen.

  3. Wählen Sie den Datensatz aus, den Sie entfernen möchten, und klicken Sie auf die Schaltfläche Entfernen.

  4. Klicken Sie auf Ja.
