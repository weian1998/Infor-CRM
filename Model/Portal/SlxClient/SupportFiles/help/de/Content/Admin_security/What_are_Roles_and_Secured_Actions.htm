Was sind Rollen und gesicherte Aktionen?

Rollen bestimmen, auf welche Funktionalität Benutzer im Web Client zugreifen können. Zu Rollen gehören gesicherte Aktionen, die bestimmen, auf welche Benutzeroberflächenelemente ein Benutzer zugreifen darf, z. B. Seiten, Menüs, Menüpunkte, Symbolleisten und Schaltflächen. Durch die Steuerung des Zugriffs auf einzelne Benutzeroberflächenelemente können Sie den Zugriff auf Aktionen wie Bearbeiten, Löschen oder Drucken kontrollieren. Derzeit gesicherte Elemente sind alle Benutzeroberflächenelemente, die Zugriff auf Firmen, Kontakte und Verkaufschancen gewähren.

Sie sichern ein Benutzeroberflächenelement ab, indem Sie dessen Eigenschaft Angewandte Sicherheit in Application Architect eine abgesicherte Aktion zuweisen. Nachdem eine gesicherte Aktion einem Benutzeroberflächenelement zugewiesen wurde, können Benutzer nur dann das dazugehörige Element anzeigen und nutzen, wenn ihnen eine Rolle zugewiesen wurde, die die gesicherte Aktion enthält. Entitäten/Firma/Speichern ist beispielsweise die gesicherte Aktion, die der Eigenschaft Angewandte Sicherheit für die Schaltflächenaktion Speichern in der Firmendetailansicht zugewiesen ist. Wenn die Schaltfläche zur Laufzeit geladen wird, ändert sich der Anzeigewert abhängig davon, ob der Benutzer die Berechtigung zum Ausführen der Aktion hat.

Ein im Web Client gesichertes Benutzeroberflächenelement dient dem Einschränken des Zugriffs aller Benutzer, bis dem Benutzer eine Rolle zugewiesen wird, die den Zugriff erlaubt. Zum Sichern von Elementen im Web Client mithilfe von Rollen müssen Sie Elementen gesicherte Aktionen hinzufügen, die gesicherten Aktionen einer Rolle hinzufügen und Benutzer der Rolle zuweisen. Elemente werden in Application Architect gesichert, indem ihnen eine gesicherte Aktion zugewiesen wird oder Sicherheitseinstellungen angewendet werden. Der Wert der Eigenschaft Angewandte Sicherheit ist der Name der gesicherten Aktion. Zum Absichern eines Elements auf der Benutzeroberfläche und Gewähren des Benutzerzugriffs auf dieses Element müssen Sie die folgenden Schritte ausführen:

  1. Legen Sie fest, mit welcher gesicherten Aktion Sie das Element schützen möchten. Erstellen Sie ggf. eine neue gesicherte Aktion im Web Client.
  2. Fügen Sie eine gesicherte Aktion zur Eigenschaft Angewandte Sicherheit des Elements in Application Architect hinzu. Weitere Informationen finden Sie in der Application Architect-Hilfe im Thema "Grundlegendes zu gesicherten Aktionen".
  3. Fügen Sie die gesicherte Aktion für dieses Element zu einer Rolle im Web Client hinzu.
  4. Weisen Sie der Rolle Benutzer zu.

Standardrollen

 Standardmäßig wird im Web Client erstellten Benutzern die Rolle "Standardbenutzer" zugewiesen. Diese Rolle erlaubt den Zugriff auf alle nicht administrativen Funktionen im Web Client. Wenn Sie einen Benutzer aus dieser Rolle entfernen, hat er keinen Zugriff auf gesicherte Elemente, es sei denn, Sie weisen ihn einer neuen Rolle zu.

 

Die Standardrollen sind:

  • Administrator - Ermöglicht das Hinzufügen, Bearbeiten und Löschen von Berechtigungen für Verwaltungsfunktionen, z. B. Benutzer, Teams, Abteilungen, Auswahllisten, Produkte, Produktpakete und die Optionen Nach Duplikaten suchen, Duplikate verarbeiten und Historie der Duplikatsuche. Dient außerdem dazu, die Bearbeitungsberechtigung für QuickForms im Web-Formular-Designer zu erteilen.
  • Datenqualitätsmanager - Ermöglicht das Hinzufügen, Bearbeiten und Löschen von Berechtigungen für Nach Duplikaten suchen, Duplikate verarbeiten und Historie der Duplikatsuche.
  • Standardbenutzer – Ermöglicht das Anzeigen, Hinzufügen, Bearbeiten und Löschen von Berechtigungen für nicht administrative Web Client-Entitäten wie Firmen, Kontakte, Verkaufschancen, Verkaufsaufträge, Interessenten, Verträge, Fehler und Tickets. Ermöglicht auch das Anzeigen von Berechtigungen für Produkte und Pakete.
    Wenn Benutzer vorhanden sind oder Sie neue Benutzer in Administrator erstellen, müssen Sie die Benutzer der Rolle Standardbenutzer zuweisen.
  • Auftrags- und Angebotsverarbeitung - Ermöglicht das Hinzufügen und Anzeigen von Berechtigungen zu Kontakten und Firmen sowie das Hinzufügen, Bearbeiten und Löschen von Berechtigungen für Verkaufsaufträge.
  • Integration - Ermöglicht das Hinzufügen, Bearbeiten, Löschen und Anzeigen von Berechtigungen für Integrationen.

    Diese Rolle wird für Standardbenutzer nicht empfohlen. Die Möglichkeit, Die Sync-Historie anzuzeigen, gehört zur Standardbenutzerrolle. Benötigen Benutzer Zugriff auf andere Integrationsaktionen, zum Beispiel die Möglichkeit, eine Integration zu autorisieren, Infor CRM empfiehlt fügen Sie diese Aktionen der Standardbenutzerrolle hinzu oder erstellen Sie eine neue Rolle, die diese Aktionen enthält.

  • Integration Contract - Preis sperren - Ermöglicht das Sperren von Produktpreisen auf einem Verkaufsauftrag. (Nur Buchhaltungsintegration)
  • Sicherheitseinstellungen im Network Client werden mithilfe von Einstellungen für Features und Funktionen sowie von administrativen Rollen bestimmt. Für die Websicherheit werden alle Network-Elemente in einem Sicherheitsmodell mit dem Namen Rolle kombiniert. Sie können das Network-Sicherheitsmodell nicht für Benutzer verwenden, die beim Web Client angemeldet sind, und das Websicherheitsmodell nicht für Benutzer verwenden, die beim Network Client angemeldet sind.
  • Benutzer mit Administratorrechten könnten ungeachtet der gesicherten Aktionen auf alle Benutzeroberflächenelemente zugreifen.
