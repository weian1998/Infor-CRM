Unterstützen einer mehrsprachigen Umgebung

Zum Vereinfachen der Lokalisierung werden Textzeichenfolgen von Steuerelementen, z. B. Beschriftungen und QuickInfos in einer Ressourcendatei gesammelt. Installationen, die mehrere Sprachen unterstützen, weisen für jede Sprachkultur eine Ressourcendatei mit übersetzten Zeichenfolgen auf. Beispiel: Eine Infor CRM-Installation, die die Basiskultur und Deutsch (in Deutschland) unterstützt, hat eine Ressourcendatei für Invariant (die Basiskultur) und eine für de-DE. Ein Web Client-Benutzer kann die deutschen Zeichenfolgen anzeigen, indem die Spracheinstellung im Browser auf de-DE festgelegt wird.

Wenn Ihre Website für die Unterstützung mehrerer Sprachen konfiguriert ist, können Sie Beschriftungen und andere Zeichenfolgen in jeder Sprache bearbeiten.

So lokalisieren Sie Textzeichenfolgen

  1. Bearbeiten Sie ein Formular mit dem Formular-Designer.
  2. Wählen Sie die Kultur aus der Dropdown-Liste.
    Sie können Kulturen zu dieser Liste hinzufügen. Weitere Informationen finden Sie in der Application Architect-Hilfe im Thema "Hinzufügen einer Kultur zur Liste in Formular-Designer".
  3. Ändern Sie Eigenschaftenwerte lokalisierbarer Steuerelementen wie Beschriftung und QuickInfo.
  4. Klicken Sie auf Speichern.
    Die Beschriftungen und andere lokalisierbare Zeichenfolgen werden für die aktuelle Kultur in der Ressourcendatei gespeichert.
  5. Zeigen Sie das Formular an. Um die lokalisierten Beschriftungen anzuzeigen, ändern Sie die Spracheinstellung im Browser.
