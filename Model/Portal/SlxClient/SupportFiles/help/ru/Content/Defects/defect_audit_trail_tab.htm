Закладка Журнал/Аудит карточки Дефекта

На данной закладке отображается история обработки карточки Дефекта. Система автоматически помещает часть информации в поля Журнала на данной закладке; эту информацию пользователи изменять не могут. Пользователь службы поддержки или менеджер может использовать закладку Журнал для отслеживания и оценки времени и действий для работы с клиентом.

Для того чтобы открыть данную закладку, откройте карточку Дефекта и выберите закладку Аудит.

В верхней части закладки отображается основная информация о выбранном Дефекте, а в разделе Аудит содержатся записи о всех действиях по работе с Дефектом. Чтобы отсортировать информацию в списке, можно нажать на заголовок колонки и использовать стрелку По возрастанию или По убыванию.

В таблице ниже описывается информация, отображаемая на этой закладке.

Поле/Колонка Пользователь Может Изменить Информацию Предоставленная Информация
Сообщено

Да - Нажмите кнопку Поиск для изменения пользователя по умолчанию.

Нажмите кнопку Календарь для изменения даты.

 Имя пользователя, получившего информацию о дефекте, а также дата получения.
Завершено

Да - Нажмите кнопку Поиск , чтобы изменить пользователя по умолчанию.

Нажмите кнопку Календарь для изменения даты.

 Имя пользователя, закрывшего заявку и дата закрытия.
Закреплено Да- Нажмите кнопку Поиск , чтобы изменить пользователя по умолчанию. Пользователь или команда пользователей, обладающие правами доступа к информации о дефекте. Информацию о доступе можно изменить на данной закладке или на карточке Дефекта.
Создал Нет Имя пользователя, который создал запись по Дефекту в системе.
Изменено Нет Имя пользователя, который последним обновил информацию о Дефекте, также указывается дата обновления.
Обновлено Нет Имя пользователя, который изменил информацию о Проблеме или ей Решении, также указывается дата обновления данных.

Представление содержит следующие колонки:

  • Дата создания - Дата создания записи в журнале.
  • Пользователь - Имя пользователя, который работал с записью по Дефекту.
  • Поле изменено - Названия измененных полей.
  • Старое значение - Колонка отображает информацию, которая содержалась в поле до изменения. Новое значение - Новое значение поля.
  • Заметки - Краткое описание изменений.
  • Значение в поле формируется автоматически.
