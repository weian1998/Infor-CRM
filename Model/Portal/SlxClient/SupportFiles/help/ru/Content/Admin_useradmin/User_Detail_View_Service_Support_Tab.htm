Закладка Сервис/Поддержка

Используйте вкладку Сервис/Поддержка на детальном представлении пользователя, чтобы установить настройки, связанные с опциями сервиса клиентов, опциями поддержки и Быстрого Поиска. При установке доступа пользователя учитывайте следующее:

  • При добавлении нового пользователя "Доступный для вызовов" - единственный параметр, выбранный по умолчанию.
  • Изменения вступают в силу при загрузке приложения. При обновлении или входе в систему, система принимает изменения.

  • При просмотре закладки Сервис/Поддержка в профиле удаленного пользователя, Ticket ID Prefix удаленного пользователя также отображается (только чтение)

Узнать больше информации о Ticket IDs

Для поддержки синхронизации между удаленными базами данных и основной (Хост) базой данных, система должна быть уверена, что создаются уникальные ID заявки, когда пользователи добавляют заявки в основном офисе, удаленных офисах или с компьютеров удаленных пользователей. Система приписывает Префикс Заявки каждой базе данных. Префикс состоит из двух частей:

  • Идентификатор места – первые три символа префикса определяют местоположение. Идентификатор места для Хост базы данных 001. Базам данных удаленных офисов приписываются идентификаторы 100-199. Базам данных удаленных пользователей приписываются идентификаторы места 200 и более. (Идентификаторы места не обязательно приписываются последовательно).
  • Key Base – последние две цифры префикса представляют номер базы данных. При создании новой базы данных для удаленного офиса или пользователя, эта цифра увеличивается.

Когда пользователь создает заявку, система приписывает корректный префикс, за которым следует 6-значный автоматически генерируемый суффикс (например, 001-01-000015).

Для настройки параметров

  1. В Детальном представлении пользователя откройте закладку Сервис/Поддержка.
  2. Укажите одну из следующих Настроек:
  • Доступный для вызовов - позволяет другим пользователям приписывать заявки этому пользователю.

Заявки, записанные на недоступного пользователя, останутся не записанными.

  • Информировать пользователя о новой заявке - отправляет e-mail сообщение, когда заявка приписывается пользователю.

Система отправляет e-mail сообщение, если приписывается новая заявка или существующая заявка перенаправляется на этого пользователя.

  • Пользователь может гарантировать доступ к клиентскому порталу - позволяет пользователю предоставлять доступ клиентам через Клиентский портал.
  • Пользователь имеет стандартно "Вставить для быстрого поиска" - устанавливает Вставить для быстрого поиска как параметр по умолчанию, когда пользователь создает новую заявку.
  • Пользователь может одобрить вставку для Быстрого поиска - позволяет пользователю определить, какие заявки и дефекты доступны через быстрый поиск.

Этот параметр доступен только в случае если возможен процесс подтверждения для быстрого поиска. Этот параметр доступен в Администраторе.

  • Уведомлять пользователя о новом дефекте - отправляет e-mail сообщение, когда дефект приписывается пользователю.

Отправляется e-mail сообщение, если пользователю назначается новый дефект, или существующий дефект переназначается на пользователя.

  1. Нажмите Сохранить.
