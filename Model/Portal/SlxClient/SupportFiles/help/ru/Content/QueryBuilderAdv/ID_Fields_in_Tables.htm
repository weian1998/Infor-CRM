Поля Идентификаторов в Таблицах

Каждая таблица содержит как минимум одно поле ИД, которое используется для обеспечения уникальности каждой записи в таблице. Когда запись добавляется в таблицу СУБЪЕКТ, значение автоматически заносится в поле ИД Субъекта. Каждый раз при добавлении записи, генерируемое значение основывается на информации в файле ключа.

акие из полей являются важными полями ИД таблицы, зависит от типа связи таблицы с другими таблицами. Любая созданная таблица, не связанная с другой таблицей, будет иметь уникальное для себя поле ИД. Если новая таблица создается как подчиненная таблица отношения Один к Одному с другой таблицей, то ее поле ИД будет полем ИД родительской таблицы. Если дочерняя таблица имеет отношение Один ко Многим с другой таблицей, ей необходимо более одного поля для обеспечения уникальности. Таким образом, будет сгенерировано два ИД при создании таблицы: один уникальный для таблицы, и один для поля ИД таблицы на которой она основана. Для связывания таблиц отношением Многий ко Многим необходимо три поля ИД. Infor CRM автоматически создаст только два ИД, одно уникальное для связанной таблицы и одно для основной таблицы. Вы должны добавить поле ИД для дочерней таблицы при создании таблицы, или не будет связи между родительской и дочерней таблицами.

В то время как существующие в Infor CRM связанные таблицы имеют два поля ID, вам необходимы все три поля ID во всех создаваемых вами связанных таблицах.
