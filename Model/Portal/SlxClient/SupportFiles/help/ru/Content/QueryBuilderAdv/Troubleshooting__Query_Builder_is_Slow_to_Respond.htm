Решение проблем: Конструктор Запросов Реагирует Слишком Медленно

 Только для Системного Администратора

Если Конструктор Запросов тратит на ответ очень много времени, то в нем возможно существуют рекурсивные связи, которые можно удалить. Пример рекурсивной связи:

Связь 1: Субъект ->Субъект 2(дополнительные поля)

Связь 2: Субъект 2(дополнительные поля) ->Субъект

Для определения рекурсивных связей

  • Выполните следующий запрос:

    SELECT j1.fromtable fromtable1, j1.fromfield fromfield1, j1.totable totable1, j1.tofield tofield1

    FROM sysdba.JOINDATA j1, sysdba.joindata j2 where j1.fromtable = j2.totable

    AND j1.totable = j2.fromtable

    AND j1.fromfield = j2.tofield

    AND j1.tofield = j2.fromfield

Не удаляйте следующие системные связи:

  • Opportunity_Salesstep.Salesstepid -> Salesstep.Salesstepid
  • Salesstep.Salesstepid -> Opportunity_Salesstep.Salesstepid

Для получения подробной информации о выполнении SQL-запросов обратитесь к Справке Администратора.
