Менеджер Глобальных Связей

 Только для Системного Администратора

  • Пользуйтесь созданием глобальных связей предельно осторожно, поскольку можно создать глобальную связь, которая создаст проблемы в работе системы.
  • По умолчанию, таблица ADDRESS соединена с таблицей ACCOUNT через Inner join. Не изменяйте параметры этого соединения.

Менеджер Глобальных Связей показывает все глобальные связи в вашей базе данных Infor CRM, и позволяет системному администратору или пользователю с правами администратора добавлять, изменять и удалять глобальные связи в случае необходимости. Глобальная связь является постоянной и существует для каждого сеанса в структуре дерева данных Конструктора Запросов.

Infor CRM содержит ряд предопределенных глобальных связей (называемых системные связи), которые не могут быть удалены. Любой может использовать эти предопределенные глобальные связи, выбрав поле в Конструкторе Запросов и нажав кнопку Глобальные Связи. Если предопределенная связь существует для этого поля, опция для ее просмотра отображается в меню как "Глобальная связь к" с именем таблицы.

Менеджер Глобальных связей доступен, только если вы зашли в систему как Администратор.

Как выполнить?

Открытие Менеджера Глобальных Связей
  1. Откройте Конструктор Запросов.
  1. Откройте Конструктор Запросов, а затем нажмите кнопку Глобальные Связи.

Эта кнопку появляется только в том случае, если вы вошли как Администратор.

В Менеджере Глобальных Связей отображаются следующие поля:

Поле Описание
Sys Системная связь, которая не может быть удалена.
1:1 Означает отношение один к одному.
Родительская Таблица, Родительское Поле Основная таблица и поле в отношении связи.
Дочерняя Таблица, Дочернее Поле Таблица и поле, связанные с Родительской Таблицей и Родительским Полем.
Тип Тип объединения, такой как Внутреннее, Левое или Правое.
Каскад Показывает, как удаляются записи в связи.
Видимость Показывает, как отображается связь в Конструкторе Запросов.
