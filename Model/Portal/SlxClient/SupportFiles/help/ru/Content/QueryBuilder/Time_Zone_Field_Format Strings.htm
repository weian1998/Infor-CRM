Строки форматирования поля Временной пояс

Приведенная ниже таблица описывает параметры строки форматирования для поля Временной пояс. Допустимые форматы строк зависят от выбранного вами типа формата.

Если никакая строка форматирования не определена для поля Временной пояс, значение форматируется с использованием отображаемого названия для соответствующей временной зоны. Например: (GMT-06:00) Central Time (US & Canada).

Строка форматирования должна начинаться со знака % и заканчиваться символом типа форматирования:

[%] [тип]
Параметр Описание

B

 Значение форматируется с помощью смещения для соответствующей временной зоны (например, 420). Смещение - это разница в минутах между средним временем по Гринвичу (UTC) и временным поясом.
DB начение форматируется с помощью смещения дневного света для соответствующей временной зоны (например, -60). Смещение дневного света - это разница в минутах между стандартным временем и летним временем.
SB Значение форматируется с помощью стандартного смещения для временной зоны (например, 0). Это значение используется во время переводов местного времени. Это значение обычно 0.
SN Значение форматируется с помощью стандартного названия для соответствующей временной зоны (например, Central Standard Time). Это значение обычно совпадает со значением, которое хранится в поле TIMEZONE.
DN Значение форматируется с помощью летнего названия для соответствующей временной зоны (например, Central Daylight Time). Если временной пояс не придерживается определенных правил, тогда значение будет отформатировано как "N/A".
