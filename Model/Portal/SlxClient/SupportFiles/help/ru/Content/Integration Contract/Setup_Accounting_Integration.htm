Установить Интеграцию учета

Эта тема больше не применяется. Для получения информации о подключении к интеграции, см. в разделе "Использование детального представления Интеграции".

Используйте диалог Установить интеграцию учета, чтобы определить детали учетной информации.

Поле Описание/Действие
Канал учета

Введите адрес портала SData, который используется Infor для обмена информацией между системами. Выполните одно из следующих действий:

  • Если ваша установка не использует порт по умолчанию (порт 80), используйте формат http://server:port/sdatavirtualdir/application/contract/operatingcompany/-/
  • Если ваша установка использует порт 80, не указывайте номер порта в адресе. Сервис Sync автоматически удаляет порт по умолчанию из адреса, что вызывает соответствующие проблемы и ошибки. Избежать этого можно, используя формат http://server/sdatavirutaldir/application/contract/operatingcompany/-/

Измените следующее:

  • Вместо application введите имя учетного приложения. Например, Sage1000.
  • Вместо contract введите gcrm.
  • Вместо operatingcompany введите компанию в вашем учетном приложении, с которой вы интегрируетесь.
Имя Пользователя Введите имя пользователя, используемое для входа в учетную систему.
Пароль Введите пароль, соответствующий имени пользователя.
Проверка канала Нажмите, чтобы проверить и отобразить статус соединения.
Статус Отображает результаты проверки снабжения учета.
Название Введите отображаемое имя для системы учета. Это имя будет видно в Веб клиенте, когда пользователи связывают записи между Infor и учетной системой.
Разрешить синхронизацию, линковку и транзакции Проставьте или снимите флажок. Когда параметр отключен, система учета не доступна для связывания или синхронизации с Infor.
Включение проверки подлинности Email

Выберите эту опцию, чтобы использовать проверку подлинности email для просмотра или проверки ограничений данных, которые может просматривать пользователь. Некоторые системы учета могут потребовать включения проверки подлинности email.

После включенмя этого параметра, вы должны изменить каждый профиль безопасности пользователей, чтобы включить адрес электронной почты, который соответствует адресу электронной почты пользователя в системе учета.
Фильтр по ресурсам

Выберите тип ресурса и нажмите кнопку Выполнить, чтобы отобразить различия схемы в таблице.

Эта функция сравнивает схемы выбранных ресурсов в двух интегрированных системах и возвращает любые различия в длине строк. Если значения строк не совпадают, используйте Database Manager или другое приложение, чтобы изменить схемы в одной из систем.

Если это не первая система учета для связи, необходимо перезапустить IIS на веб-сервере, для того чтобы новые каналы были включены во время синхронизации.
