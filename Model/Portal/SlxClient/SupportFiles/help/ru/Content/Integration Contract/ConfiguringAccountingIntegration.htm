Настройка Интеграции Учета Х3

Выполните следующие задачи для настройки Интеграции учета. Х3

  1. Если работает поддержка мультивалютности, убедитесь, что Infor и ваша учетная система имеют одинаковую основную валюту.

  2. Запустите SLXConversionUtility.exe.
    Необходима поддержка одной или всех интеграций.

    		3.

    Проверка подлинности Windows должна быть настроена для пользователя с правами администратора перед запуском Утилиты обновления телефонов и E-mail. Дополнительные сведения см. в разделе «Использование проверки подлинности Windows» в разделе справки Администратора.

  3. Настройте синхронизациюInfor SData.
    Необходима поддержка одной или всех интеграций.

  4. В списочном представлении интеграции выберите X3.

  5. Просмотрите информационные поля.

  6. В поле Тип ссылки убедитесь, что выбрано Система.

  7. Установите Включить.

  8. Настройте синхронизацию.

    1. В поле Каждая синхронизациявведите количество минут, которое должно пройти между началом циклов синхронизации.
    2. В поле Запуск синхронизации между нажмите кнопку кнопку Календарь и выберите время начала и время завершения периода, в течение которого синхронизация должна произойти. Infor рекомендует выбирать период, который не приходится на полночь.
  9. Нажмите Сохранить
  10. На закладке Ссылки нажмите кнопку Добавить ссылку.
    1. В поле URL, введите SData URL, который Infor использует для обмена информацией между системами.
      • Воспользуйтесь одним из следующих форматов:
        • Если ваша реализация не использует порт по умолчанию (порт 80), используйте формат: http://server:port/sdatavirtualdir/application/contract/dataset/
        • Если ваша реализация использует порт 80, не указывать номер порта из URL.
          Синхронизация служба автоматически удаляет порт по умолчанию из URL, который вызывает соответствующие вопросы и ошибки. Избежать этого можно, используя формат: http://server/sdatavirtualdir/application/contract/dataset/
      • Замените значения, предоставляемые партнером по интеграции:
        •  Sdatavirtualdir - Например, sdata.

        •  application - наименование системы учета. Например, accounts50.

        •  contract - имя контакта. Например, GCRM.
        •  dataset - различает набор данных или рабочей компании. Например, / - /
    2. В поле Отображаемое имя введите краткое, но понятное наименование. Г
    3. В поле Имя пользователявведите имя пользователя для URL.
    4. В поле Пароль введите пароль для URL.

    5. Нажмите кнопку Тестировать ссылкуи просмотрите информацию в поле Статус.
      Если канал протестирован не успешно, ознакомьтесь с информацией о ссылке, внесите необходимые изменения и проверьте ссылку снова.

    6. Нажмите OK.
  11. Сопоставьте схему БД Infor со схемой источника интеграции для обеспечения целостности данных.

    12. Если вы создавали пользовательские сущности, которые не соответствуют Infor, синхронизация может не работать.
  12. Воспользуйтесь закладкой Опции для специальных настроек X3.

    1. Рядом с каждой опции, нажмите в столбце Значение и введите значение True, чтобы включить или False, чтобы отключить

      • AccountingSystemManagesSO - Если True, удаляется возможность создать и отредактировать заказы на продажу в Infor. Должна использоваться система учета, чтобы управлять заказами на продажу.

      • EnableEmailAuthentication - Если True, то проверки подлинности электронной почты используется для ограничения данных, которые можно увидеть или проверить, что пользователь просмотрел. Некоторым системам учета может требоваться проверка подлинности электронной почты. После включения данной опции, необходимо изменить профиль каждого пользователя, чтобы включить адрес электронной почты, который соответствует адресу электронной почты пользователя в системе учета.
      • RestrictToSingleAccountingSystem - Если True, ограничивает вашу интеграцию Infor с одной системой учета.
    2. Нажмите Сохранить
  13. В представлении Конфигурации Синхронизации SData щелкните на закладке Совпадения.
    1. Нажмите Изменить рядом с ресурса, чтобы установить глобальные критерии соответствия по умолчанию для пользователей создания ссылок.
    2. Выберите Свойства и Оператордля критериев сравнения.
    3. Для добавления критериев сравнения нажмите .
    4. После того как закончите нажмите OK.
  14. Повторите шаг 13 для каждого ресурса.

  15. Настройте синхронизацию службы Интеграции. Необходима поддержка одной или всех интеграций.

  16. (Если необходимо), экспорт и слияние записей из вашей системы учета.
  17. Настройте роли пользователя для интеграции.

    • Роль интеграции блокирования цены контракта дает разрешение, чтобы зафиксировать цены продукта на заказ клиента.

    • Действия продвижения субъектов предоставляет пользователям права для продвижения субъектов и является частью роли обычного пользователя по умолчанию.
