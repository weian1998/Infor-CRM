Пользователь возвращается после длительного отсутствия

Только системный администратор и пользователи с соответствующей ролью могут иметь доступ к этим функциям.

Когда пользователь возвращается после длительного отсутствия, воспользуйтесь контрольным списком пользователь отсутствует длительное время, чтобы убедиться, что вы выполнили все необходимые действия.

Этот список использует те же сценарии. В первую очередь завершите пункт Все сценарии, затем обратитесь к дополнительным пунктам сценария, подходящим для вашей организации.

Все сценарии

  • Если вы запретили пользователю, отсутствовавшему долгое время, входить в систему в течение его отсутствия, откройте детальное представление пользователя и проставьте флаг Логин доступен.

СЦЕНАРИЙ A - Субъекты принадлежат командам, отсутствующий пользователь и пользователь-заместитель в одних и тех же командах.

Никаких специальных шагов в этом сценарии предпринимать не надо. Отсутствовавший пользователь также имеет доступ к тем же субъектам, что и пользователь-заместитель, поскольку они в одной команде.

СЦЕНАРИЙ B - Субъекты принадлежат командам, но отсутствующий пользователь и пользователь-заместитель в разных командах.

  • Просмотрите свои заметки, когда вы добавляли пользователя-заместителя к необходимым командам, удалите пользователя-заместителя из нужных команд.

СЦЕНАРИЙ C - Субъекты принадлежат индивидуальным пользователям.

  • Откройте детальное представлении пользователя для отсутствующего пользователя.
  • Выберите закладку члены команды пользователя. Нажмите ссылку Удалить для пользователя-заместителя.

Удаление этого человека из профиля прекращает его доступ к субъектам, относящимся к отсутствующему пользователю.

СЦЕНАРИЙ D - Ваша организация использует группу "Все" для определения принадлежности субъектов.

Никаких специальных шагов в этом сценарии предпринимать не надо. Отсутствовавший пользователь имеет доступ ко всем субъектам, если ваша компания использует группу Все для определения принадлежности субъектов.
