Совпадения между записями интеграции

Чтобы избежать повторяющихся записей и найти соответствующие записи в одном из приложений, поиск осуществляется на основе следующих ключевых полей:

Адресная книга / Контакты Субъекты

Назначение / активность

 

 Дела
e-mail адрес (EMAIL1, EMAIL2, EMAIL3)

Субъект/Компания

 Дата/Дата начала Дата/Дата начала
 

 

 Описание/ Тема Описание/ Тема
    Продолжительность Продолжительность

Ключевые поля чувствительны к регистру. Например, если у вас есть запись для Lee Hogan в Infor и один для Lee Hogan в Microsoft Outlook, записи не идентичны. Обе записи добавляются в каждую систему во время синхронизации.

Сопоставление контактов

Контакты согласованы с помощью электронной почты. Если совпадение не найдено, то создается новая запись. Чтобы не создавать дубликаты записей рекомендуется проверить, что контактные записи для синхронизации имеют уникальный адрес электронной почты для обоих приложений.

Если точное соответствие адреса электронной почты не может быть найдено, то происходит следующее:

  • В Infor, если адрес электронной почты не может быть найден, то создается новый контакт.
  • В Infor при наличии нескольких записей:
    • Infor пытается сопоставить с помощью фамилию контакта и имя.
    • Если совпадение не может быть найден тихо, то новый контакт создается в Infor.

  • В интегрированном приложении, если у многократных записей есть тот же адрес электронной почты, то контакт соединен с первым контактом, найденным с соответствующим адресом электронной почты.
