Детальное представление Истории синхронизации

Используйте Детальное представление Истории синхронизации, чтобы просматривать подробную информацию об истории синхронизации. Элементы истории синхронизации связаны с заданиями, выполненными между Infor и внешним источником интеграции.

  • Чтобы сортировать сетку по определенному полю, нажмите на один из заголовков столбцов.
  • Чтобы вывести на экран элементы истории, которые соответствуют определенным критериям, настройте фильтры.

Как выполнить?

Открыть Детальное представление Истории синхронизации
  • На навигационной панели нажмите Администрирование и выберите История синхронизации , чтобы открыть списочное представление Истории синхронизации.

Используйте Поиск для нахождения записей

Переключиться в режим списочного представления
  • На панели инструментов нажмите кнопку Список.

Что такое группы?

Информационные поля 

Информационные поля содержат основную информацию об элементе синхронизации.

Поля Описание
Метка даты времени Дата и время начала задания синхронизации
Тип ресурса Тип ресурса синхронизируемого между приложениями.
Ошибочная последовательность Выводит на экран последовательность числа, если многократное сообщение и результаты соединены от одного задания. Например, если одно синхронизирующее задание произведет 3 ошибки, то Вы будете видеть запись для ошибок 1, 2, и 3.
Синхронизирующее задание Синхронизация, которая содержит синхронизирующий элемент истории.
Http статус Состояние текущего цикла синхронизации.
Путь полезной нагрузки Выражение XPath, которое относится к элементу полезной нагрузки, ответственному за ошибку. Этот элемент только используется, когда ошибка связана с определенной частью данных, отправляемых поставщику услуг. Например, при обновлении ресурса или представлении запроса на обслуживание.
Полученный приложением Приложение, которое получило изменение.
Отправленный приложением Интеграция используется для передачи изменений.
Код приложения Специализированный код диагноза.
Уровень важности

Уровень важности диагноза. Возможные значения:

  • Информация - Никакое особое внимание не требуется.
  • Предупреждение - работе не препятствуют успешно выполниться, но может потребовать внимания.
  • Переходный процесс - работа перестала работать, но может успешно выполниться позже при том же условии.
  • Ошибка - работа перестала работать и должна быть изменена перед представлением.
  • Фатальный - работа содержит фатальную ошибку и должна быть изменена перед представлением. Другие операции могут также перестать работать.
Код неисправности

SData код неисправности. Ниже приведены допустимые коды диагностики:

  • BadUrlSyntax- Синтаксис URL - недействителен.
  • BadQueryParameter - Параметр запроса - недействителен.
  • ApplicationNotFound - Приложение не существует.
  • ApplicationUnavailable- Приложение существует, но не доступно.
  • DatasetNotFound- Набор данных не существует.
  • DatasetUnavailable- Набор данных существует, но не доступен.
  • ContractNotFound- Контракт не существует.
  • ResourceKindNotFound- Вид ресурса не существует.
  • BadWhereSyntax- у условия есть недопустимый синтаксис.
  • ApplicationDiagnosisСпециализированный диагноз. Детали выводятся на экран в столбце Код приложения.
Сообщение об ошибке Полное сообщение об ошибке.
Трассировка стека Список классов и методов, к которым получили доступ до ошибки. Информация выводится на экран, если была ошибка во время синхронизации.
Детали полезной нагрузки

Полный XML полезной нагрузки. Эта информация сохранена только, когда есть ошибка при применении ее.
