Onglet Actions

Utilisez l'onglet Actions de la vue Détails du rôle pour gérer les actions sécurisées affectées un rôle. Un rôle détermine les fonctionnalités auxquelles un utilisateur a accès dans le client Web. L'ajout d'actions sécurisées à un rôle détermine les fonctionnalités auxquelles le rôle permet d'accéder. L'ajout d'utilisateurs au rôle détermine à qui l'accès est autorisé.

En savoir plus...

Les éléments sont sécurisés dans l'application Architect en leur attribuant une action ou en appliquant une sécurité. Une fois qu'une action sécurisée est appliquée, les utilisateurs ne peuvent accéder à cet élément que s'ils sont affectés à un rôle incluant l'action sécurisée. Par exemple, sélectionnez Entités/Compte/Enregistrer comme valeur de la propriété Sécurité appliquée pour l'action du bouton Enregistrer sur la vue principale Compte. Lorsque ce bouton se charge à l'exécution, la valeur d'affichage change selon que l'utilisateur actuel possède ou non l'autorisation d'effectuer l'action.

Pour plus d'informations, consultez la rubrique « Comprendre les actions sécurisées » dans l'aide de l'application Architect.

L'ajout d'une action sécurisée à un rôle, ou sa supression, à l'aide de cet onglet ajoute également l'action sécurisée à l'onglet Rôles dans la vue Détails des actions sécurisées, ou supprime l'action sécurisée de cet onglet.

Procédure

Ajout d'une action sécurisée à un rôle

  1. Ouvrez la vue Détails du rôle.
  2. Cliquez sur l'onglet Actions.
  3. Cliquez sur Ajouter une action.
  4. Utilisez la Recherche pour localiser et sélectionner les actions que vous souhaitez ajouter au rôle.
  5. Cliquez sur Ajouter les actions sélectionnées.

Supprimer une action sécurisée d'un rôle

  1. Ouvrez la vue Détails du rôle.
  2. Cliquez sur l'onglet Actions.

  3. Sélectionnez l'enregistrement à retirer, puis cliquez sur le bouton Supprimer.

  4. Cliquez sur Oui.
