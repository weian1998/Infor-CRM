Opérateurs d'instruction de Query Builder

Lorsque vous ajoutez une instruction conditionnelle, vous devez sélectionner un opérateur. L'opérateur permet de déterminer si un enregistrement répond ou non aux critères définis dans une instruction conditionnelle. Lorsque vous ajoutez une instruction conditionnelle, l'opérateur par défaut et les autres opérateurs disponibles dépendent du type de champ.

Le tableau suivant décrit tous les opérateurs possibles.

Opérateur Sélectionne un enregistrement si le champ correspondant contient une valeur :
Commençant par qui commence par la valeur définie dans la zone La valeur est.
Contient qui contient la chaîne définie dans la zone La valeur est.
(=) égal à égale à la valeur définie dans la zone La valeur est.
(<>) non égal à non égale à la valeur définie dans la zone La valeur est.
(<) inférieur à inférieure à la valeur définie dans la zone La valeur est.
(<=) inférieur ou égal à inférieure ou égale à la valeur définie dans la zone La valeur est.
(>) supérieur à supérieure à la valeur définie dans la zone La valeur est.
(>=) supérieur ou égal à supérieure ou égale à la valeur définie dans la zone La valeur est.
In qui contient une des chaînes définies dans la zone La valeur est, telle qu'une instruction OR littérale intégrée. Vous devez utiliser les guillemets pour chaque valeur indiquant ce qui doit être recherché.
Ne contient pas de données nulle (ne contient aucune valeur).
Contient des données non nulle (contient une valeur quelconque).
dans les xxx derniers jours est une date comprise dans les « xxx » derniers jours (passé). Saisissez soit un nombre entier soit deux séparés par une virgule pour une plage de dates.
dans les xxx prochains jours est une date comprise dans les « xxx » prochains jours (futur). Saisissez soit un nombre entier soit deux séparés par une virgule pour une plage de dates.
