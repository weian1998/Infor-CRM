Onglet Historique de synchronisation - Tâche de synchronisation

L'onglet Historique de synchronisation dans la vue Détails des tâches de synchronisation liste chaque enregistrement de l'historique de synchronisation qui est associé à une tâche de synchronisation.

Procédure

Ouvrir l'onglet Historique de synchronisation
  • Ouvrez la vue Détails des tâches de synchronisation, puis cliquez sur Historique de sychronisation.
Filtrer la liste de l'onglet Historique de synchronisation
  1. Cliquez sur le bouton Filtrer par nom d'utilisateur Rechercher, puis sélectionnez l'utilisateur dont vous voulez afficher les enregistrements de l'historique de synchronisation.
  2. Dans la liste Filtrer par type de ressource, sélectionnez l'entité du type de journal sélectionné.

Cette vue contient les colonnes suivantes :

  • Nom de journal de l'historique de synchronisation - ouvre la vue Détails de l'historique de synchronisation.
  • Utilisateur - nom de l'utilisateur qui a apporté la modification.
  • Ressource - type de ressource qui a été synchronisé.
  • Enregistrement - ouvre la vue Détails correspondant à l'enregistrement qui a été mis à jour.
  • Envoyé depuis - application qui a envoyé la modification.
  • Reçu par - application qui a reçu la modification.
  • Sévérité - type d'information consigné pour l'élément d'historique de synchronisation.
  • Date de création du journal - date à laquelle l'élément du journal de l'historique de synchronisation a été créé.
  • Lien - lien utilisé pour synchroniser les données entre Infor CRM et l'intégration externe.
  • Intégration - intégration impliquée dans la synchronisation des informations.
  • Etat - résultat de la synchronisation. Les états possibles incluent :
    • Créé - L'enregistrement a été créé dans l'application qui a reçu la modification.
    • Conflit gagné - L'enregistrement a été mis à jour dans l'application qui a reçu la modification résultant des règles de résolution de conflit.
    • Conflit perdu - L'enregistrement a été mis à jour dans l'application qui a reçu la modification résultant des règles de résolution de conflit.
    • Supprimé -L'enregistrement a été supprimé de l'application qui a reçu la modification.
    • Pas de modification - aucune modification n'a été apportée dans l'application qui aurait dû la recevoir en raison d'erreurs.
