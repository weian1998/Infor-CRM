Dépannage - Intégrations - Impossible de démarrer le service de synchronisation SData Infor CRM

Vous pouvez rencontrer les problèmes suivants lors de la synchronisation d'informations entre Infor CRM et les intégrations prises en charge.

Problème :

Impossible de démarrer le service de synchronisation SData Infor CRM

Solutions :

  • Vérifiez que l'authentification Windows est activée et mappée à l'utilisateur pour lequel le service doit se connecter et s'exécuter.

    Infor CRM recommande d'utiliser une licence d'accès externe pour cet utilisateur et non une licence ADMINISTRATEUR.

  • Assurez-vous que la connexion dans le fichier SLXSDataSyncServer.exe.config est correctement définie. En savoir plus...
    • Les informations de connexion doivent correspondre au fichier connection.config déployé sur votre site Web.
      Par défaut, le fichier SLXSDataSyncServer.exe.config se trouve dans ...\Program Files\Saleslogix\ ou ...\Program Files (x86)\Saleslogix\  et le fichier de connexion est déployé dans ...\inetpub\wwwroot\slxclient
  • Assurez-vous que la configuration de la synchronisation SData est configurée correctement avec les flux Infor CRM, le nom d'utilisateur et le mot de passe corrects.
  • Assurez-vous que la ou les intégrations sont correctement configurées sur la vue Détails de l'intégration.
    • Assurez-vous que la synchronisation est activée.
  • Ouvrez le fichier SLXSDataSyncServer.exe dans la fenêtre d'invite de commande pour afficher la sortie de l'application.
  • Affichez les erreurs de journal, les avertissements ou messages de débogage. En savoir plus...
    • Dans la vue Détails de la tâche de synchronisation , ouvrez l'onglet Eléments du journal et filtrez par Type de journal pour afficher les erreurs, les avertissements ou les messages de débogage.
  • Dans le serveur Web, vérifiez si le Visualiseur d'événements Windows contient des erreurs.
  • Activer la connexion supplémentaire sur le serveur Web. En savoir plus...
    1. Dans le dossier d'installation Infor CRM, créez ou ouvrez le fichier log4net.config pour sortir les messages de débogage SLXSDataSyncServer vers le Visualiseur d'événements.
    2. Ajoutez ou mettez à jour ce qui suit :
      <log4net threshold="ALL">
      <appender name="EventLogAppender"
      type="log4net.Appender.EventLogAppender" >
      <applicationName value="Services" />
      <layout type="log4net.Layout.PatternLayout">
      <conversionPattern value="%date [%thread] %-5level %logger
      [%property{NDC}] - %message%newline" />
      </layout>
      </appender>

      <logger name="SDataSync">
      <level value="ALL" />
      <appender-ref ref="EventLogAppender"/>
      </logger>
      </log4net>
    3. Dans le dossier ...\Inetpub\wwwroot\SLXClient, ouvrez le fichier log4net.config et répétez l'étape b. Ceci permet de rédiger des messages de débogage et d'information pour les processus SData initiés par le client. Par exemple, le service de tarification appelle.
    4. Dans le dossier ...\Inetpub\wwwroot\sdata, ouvrez le fichier log4net.config et modifiez la valeur de niveau SDataSync d'INFO à TOUS ou DEBOGAGE, en fonction des messages que vous voulez afficher dans l'onglet Éléments du journal.
      <logger name="SDataSync">
      <level value="INFO" />
      <appender-ref ref="EventLogAppender"/>
