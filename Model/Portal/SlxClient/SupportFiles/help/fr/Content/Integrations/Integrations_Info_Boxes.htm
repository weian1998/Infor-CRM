Zones d'informations d'intégration

Les zones d'informations d'intégration contiennent des informations sur l'intégration sélectionnée. Selon l'intégration, vous pouvez afficher ou modifier les informations si nécessaire. Certaines informations sont non modifiables.

Les informations suivantes s'affichent sur la vue Détails Intégration.

Zone Description/Action
Intégration Nom de l'intégration.

Type d'authentification

 

Méthode d'authentification des données.

La modification du type d'authentification peut causer l'échec de l'intégration.

Le service externe doit être autorisé par un ou plusieurs utilisateurs du service si la méthode est OAuth 1 ou OAuth2, que l'intégration est activée et que ID client et Secret sont spécifiés.
Méthode de synchronisation Définit comment ou si les informations sont synchronisées entre le système intégré et Infor CRM. Les options incluent Ne se synchronisent pas, Synchronisation côté client et Synchronisation côté serveur. Synchronisation côté serveur indique que le Infor CRMService de synchronisation d'intégration est utilisé.
Type de lien Type de lien utilisé pour communiquer les données entre le système intégré et Infor CRM. Si Admin ou Système est sélectionné, les données sont gérées via un seul lien. Si Utilisateur est sélectionné, les données sont communiquées pour chaque utilisateur via un lien séparé.
ID client

Types d'authentification OAuth1 ou OAuth2 seulement.

Assigné par le fournisseur d'intégration durant le processus d'enregistrement.
Secret

Types d'authentification OAuth1 ou OAuth2 seulement.

Assigné par le fournisseur d'intégration durant le processus d'enregistrement.
Activé

Utilisez cette option pour activer ou désactiver la synchronisation pour cette intégration.

Si cette option n'est pas sélectionnée, aucune donnée ne sera échangée entre le système intégré et Infor CRM. Par ailleurs, les options de synchronisation suivantes ne seront pas disponibles.
Gestionnaire de synchronisation

Ces informations sont en lecture seule.

Définit un gestionnaire de synchronisation non-SData pour une intégration qui n'expose pas ses propres points finals de synchronisation SData.

Le gestionnaire est responsable du mappage des données entre le contrat et l'intégration.
Synchroniser toutes les Définit le nombre de minutes devant s'écouler entre les cycles de synchronisation.
Synchroniser entre Définit les heures entre lesquelles l'intégration est synchronisée.
