Création d'une proposition Where

La création d'une clause Where est recommandée aux utilisateurs expérimentés seulement.

Une clause Where permet de filtrer les informations issues d'un flux SData. Le langage d'interrogation SData permet d'exprimer les conditions de filtrage avec le paramètre d'interrogation Where. Les informations issues d'un flux SData sont soumises à la sécurité ; par conséquent, les utilisateurs voient seulement les enregistrements qu'ils ont la permission de voir.

La syntaxe de création d'une clause Where est très spécifique. Si la syntaxe est incorrecte, vous pourrez avoir des résultats indésirables.

Pour créer une clause Where, vous devez connaître :

  • La ressource associée à votre widget de flux SData
  • Le nom de la colonne qui contient les données que vous voulez filtrer

    Les colonnes ne peuvent pas toutes servir à filtrer des données.

  • Les valeurs que vous recherchez
  • Ce que sont les opérateurs SData valides

  • Les valeurs de texte doivent être entre des apostrophes droites (' ')

  • Les dates doivent être entre des symboles @

    Sag

  • Un texte dans une instruction similaire doit être placé entre des apostrophes droites (' ') et des symboles %

Pour créer une clause Where

  • Dans la zone Clause Where de l'assistant widget SData Modifier les paramètres du widget, tapez une clause Where en utilisant la syntaxe appropriée. N'insérez pas de clause « where= ».
    Par exemple, NomCompte eq 'State Enterprises Co.'

Si votre widget de flux SData ne retourne pas les données comme prévu, voici quelques erreurs courantes :

  • Assurez-vous que la clause Where contient des apostrophes droites (' '), et non pas incurvées (‘ ’).
  • Assurez-vous que le nom de la colonne est correct. Les noms de colonne sont sensibles à la casse et basés sur le nom sous-jacent, et non pas sur le nom convivial qui apparaît dans les vues Infor CRM. Par exemple, NomCompte, pas Compte ou Nom de compte.
