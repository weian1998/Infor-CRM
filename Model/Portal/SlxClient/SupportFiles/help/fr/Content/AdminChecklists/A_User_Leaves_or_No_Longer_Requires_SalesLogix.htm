Un utilisateur quitte la société ou n'a plus besoin de Infor CRM

L'administrateur système et les utilisateurs dotés d'un rôle approprié peuvent accéder à ces fonctionnalités.

Si un utilisateur quitte votre société, il est préférable de convertir son profil en utilisateur désactivé plutôt que de le supprimer. Les utilisateurs désactivés n'utilisent pas de licence et ne peuvent pas se connecter, mais leur nom figure toujours dans la base de données. Ceci évite que les enregistrements d'historique ne portent la mention « Utilisateur inconnu ». Suivez la liste de vérification ci-après lors du départ d'un utilisateur.

  • Si des comptes appartiennent à des utilisateurs individuels dans votre organisation et que l'utilisateur quittant la société est le propriétaire de comptes, vous devez réaffecter ces derniers. Utilisez le réalignement de territoire dans l'Administrateur pour réaffecter les comptes d'un utilisateur à un autre utilisateur.
  • Ouvrez la vue Détails de l'utilisateur.
  • Pour l'utilisateur qui part, utilisez l'onglet Appartenance à une équipe pour déterminer si l'utilisateur est un propriétaire d'équipe. Lorsque c'est le cas, définissez le profil de sécurité de l'utilisateur sur un profil différent de propriétaire d'équipe.
  • Si la case L'utilisateur est un responsable est cochée, désélectionnez-la. (Il est impossible de désactiver un utilisateur défini en tant que responsable.)
  • Dans la zone Nom, tapez DESAC. (ou mention similaire) devant le nom de l'utilisateur. Cette étape n'est pas essentielle, mais permet d'identifier plus facilement que l'utilisateur est désactivé.
  • Dans la liste Type d'utilisateur, sélectionnez Utilisateur désactivé.
