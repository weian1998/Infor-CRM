Configuration de l'intégration du système de comptabilité X3

Procédez comme suit pour configurer l'intégration d'un système de comptabilité X3.

  1. Si la fonctionnalité de devises multiples est activée, vérifiez que Infor CRM et votre système de comptabilité utilise la même devise de base.
  2. Exécutez le fichier SLXConversionUtility.exe.
    Une fois suffit pour la prise en charge de toutes les intégrations. 

    L'authentification Windows doit être configurée pour l'utilisateur administratif avant d'exécuter l'utilitaire de conversion. Pour les étapes détaillées, consultez la rubrique « Utilisation de l'authentification Windows » dans l'aide du module Administrateur.

  3. Configurer laInfor CRM synchronisation SData.
    Une fois suffit pour la prise en charge de toutes les intégrations.
  4. Dans la vue Liste des intégrations, sélectionnez X3.
  5. Examinez les zones d'informations.
  6. Sous Type de lien, assurez-vous que Système est sélectionné
  7. Cochez la case Activé.
  8. Définissez les options de synchronisation.
    1. Dans le champ Synchroniser toutes les, tapez le nombre de minutes qui doit s'écouler entre le début des cycles de synchronisation.
    2. Dans les champs Synchroniser entre, cliquez sur le BoutonAgenda et sélectionnez l'heure de début et l'heure de fin marquant la période durant laquelle la synchronisation doit avoir lieu. Infor CRM recommande que les heures sélectionnées ne chevauchent pas minuit.
  9.  Cliquez sur Enregistrer.
  10. Sur l'onglet Liens, cliquez sur le bouton Ajouter un lien.
    1. Dans le champ URL, tapez l'URL SData utilisée par Infor CRM pour partager les informations entre les systèmes.
      • Utilisez un des formats suivants :
        • Si votre implémentation n'utilise par le port par défaut (port 80), utilisez le format http://server:port/sdatavirtualdir/application/contract/dataset/
          •  
        • Si votre implémentation utilise le port 80, omettez le numéro de port de l'URL.
          Le service de synchronisation expurge automatiquement le port par défaut de l'URL qui cause les erreurs et les problèmes correspondants. Evitez ceci en utilisant le format http://server/sdatavirtualdir/application/contract/dataset/
      • Remplacez par les valeurs fournies par votre partenaire d'intégration :
        •  Sdatavirtualdir - Par exemple, sdata.

        •  application - le nom de votre application de comptabilité. Par exemple, comptes50.

        •  contract - le nom du contrat. Par exemple, GCRM.
        •  dataset - fait la distinction entre un jeu de données et une société d'exploitation. Par exemple, /-/

    2. Dans le champ Nom d'affichage, tapez un nom bref mais descriptif.
    3. Dans le champ Nom d'utilisateur, tapez le nom d'utilisateur correspondant à l'URL.
    4. Dans le champ Mot de passe, tapez le mot de passe correspondant à l'URL.

    5. Cliquez sur le bouton Tester le lien et affichez les informations dans la zone Etat.
      Si le flux échoue au test, examinez les informations d'association, apportez vos modifications et testez une nouvelle fois le lien.

    6. Cliquez sur OK.
  11. Comparez le schéma de la base de données dans Infor CRM à votre schéma d'intégration pour garantir l'intégrité des données.

    Si vous avez personnalisé Infor CRM ou si vous avez créé des entités personnalisées dans l'intégration qui ne correspondent pas à Infor CRM, la synchronisation de ces informations ne fonctionne peut-être pas.

  12. Utilisez l'onglet Options pour définir les options spécifiques de X3.
    1. A côté de chaque option, cliquez dans la colonne Valeur et tapez Vrai pour activer ou Faux pour désactiver les options suivantes
      • AccountingSystemManagesSO - Si Vrai, empêche de créer et de modifier des bons de commande dans Infor CRM. Le système de comptabilité doit être utilisé pour gérer les bons de commande.
      • EnableEmailAuthentication - Si Vrai, l'authentification de la messagerie est utilisée pour limiter les données visibles ou pour vérifier ce qu'un utilisateur a vu. Certains systèmes de comptabilité peuvent nécessiter l'authentification de la messagerie. Après avoir activé cette option, vous devez modifier chaque profil utilisateur afin d'inclure l'adresse e-mail qui correspond à l'adresse e-mail d'un utilisateur dans le système de comptabilité.
      • RestrictToSingleAccountingSystem - Si Vrai, limite votre intégration à Infor CRM et à un système de comptabilité.
    2. Cliquez sur Enregistrer. 
  13. Dans la vue Configuration de la synchronisation SData, cliquez sur l'onglet Correspondance.
    1. Cliquez sur Modifier à côté d'une ressource pour définir des critères de correspondance globaux par défaut pour les utilisateurs créant des liens.
    2. Sélectionnez Propriété et Opérateur pour les critères de correspondance.
    3. Pour ajouter des critères de correspondance supplémentaires, cliquez sur .
    4. Lorsque vous avez terminé, cliquez sur OK.
  14. Répétez l'étape 13 pour chaque ressource si nécessaire.
  15. Configurez le service de synchronisation d'intégration.Une fois suffit pour la prise en charge de toutes les intégrations.
  16. Si nécessaire, exportez et fusionnez les enregistrements de votre système de comptabilité.
  17. Configurez les rôles de l'utilisateur pour cette intégration.
    • Le rôle Verrouillage du prix du contrat d'intégration permet d'accorder l'autorisation de verrouiller le prix d'un produit sur un bon de commande.

    • L'action PromouvoirCompte permet d'accorder l'autorisation de promouvoir des comptes et fait partie du rôle Utilisateur standard par défaut.
