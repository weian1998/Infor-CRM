Présentation de Infor CRM Client Web déconnecté

Utilisateurs distants uniquement

Le Client Web déconnecté fonctionne comme le Client distant Infor CRM si ce n'est que les utilisateurs se connectent à leur base de données locale via le client Web. Le Client Web déconnecté vous permet de travailler hors ligne sans connexion Internet active.

Les modifications entre la base données de l'utilisateur du Client Web déconnecté et la base de données du bureau principal sont transférées grâce à la synchronisation. Pour synchroniser les modifications entre votre base de données et la base de données du bureau principal, vous devez utiliser le client de synchronisation. Pour plus d'informations, reportez-vous à l'Aide en ligne du client de synchronisation.

Pour exécuter des rapports dans un état déconnecté, le Client Web déconnecté doit être exécuté sur le serveur Windows 2003 avec IIS. Le serveur Reporting Client Web Infor CRM doit également être installé et configuré. Si ces conditions ne sont pas remplies, les utilisateurs du client Web déconnectés ne peuvent exécuter Web Reporting que s'ils sont connectés au serveur Web Reporting hôte. Ces utilisateurs verront des rapports qui représentent les données de la base de données hôte et non de leur base de données distante.

Quelle est la différence dans le Client Web déconnecté ?

  • La vue des Nouveautés permet aux utilisateurs du Client Web déconnecté de s'abonner et de se désabonner des comptes.
  • (Intégration de la comptabilité uniquement) La synchronisation du contrat entre Infor CRM et un système de comptabilité pris en charge a lieu dans la base de données hôte. Par conséquent, toutes les modifications de Client Web déconnecté ou du système de comptabilité doivent être synchronisées via la base de données hôte.
