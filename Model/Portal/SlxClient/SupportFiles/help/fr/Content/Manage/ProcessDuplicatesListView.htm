Vue de liste Traiter les doublons

L'administrateur système et les utilisateurs dotés d'un rôle approprié peuvent accéder à ces fonctionnalités.

La vue de liste Traiter les enregistrements en double affiche tous les doublons potentiels identifiés par les tâches exécutées dans l'Assistant de recherche des doublons. Vous pouvez utiliser cette vue pour choisir de traiter l'enregistrement comme un doublon ou non.

Cette vue contient les colonnes suivantes :

  • Entité source - le compte dans Infor CRM qui est un doublon potentiel.
  • Type de la source : l'entité sur laquelle la tâche a été exécutée.
  • Etat : l'état de résolution. L'état de doublon potentiel indique que l'enregistrement n'a pas été traité. L'état N'est pas un doublon ou Fusionné indique que le doublon a été traité et n'est pas un doublon, ou qu'il a été fusionné avec l'enregistrement approprié.
  • Score : la probabilité que l'enregistrement soit un doublon.
  • Entité de résultat : le compte qui correspond au compte de l'entité source Saleslogixet apparaît comme doublon potentiel.
  • Type d'entité de résultat : le type d'entité de l'enregistrement.
  • Nombre de résultats : le nombre de doublons potentiels dans la tâche.
  • Numéro de la tâche : la tâche dans laquelle le doublon potentiel a été identifié.
  • Date du résultat : la date à laquelle la tâche a été exécutée.

Procédure

Ouvrir la vue de liste Historique de la recherche des doublons
  • Dans le menu Outils, cliquez sur Traiter les enregistrements en double.

Utilisez la Recherche pour rechercher les enregistrements

Ouvrir la vue Détails de l'historique de la recherche des doublons
  • Cliquez sur le numéro de tâche pour ouvrir la vue Détails de l'historique de la recherche des doublons.
