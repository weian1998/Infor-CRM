Risoluzione dei Problemi di Integrazione Contabilità - Il Servizio Sincronizzazione SData Infor CRM non si avvia

E' possibile verificare il seguente problema durante la sincronizzazione di informazioni tra Infor e le integrazioni supportate.

Problema:

Non è possibile avviare il Servizio di Sincronizzazione SData di Infor CRM

Soluzioni:

  • Verificare che l'Autenticazione Windows sia abilitata e configurata per l'utente che deve accedere al servizio ed eseguirlo.

    Infor consiglia l'utilizzo di una licenza di Accesso Esterno per questo utente e non una licenza ADMIN.

  • Verificare che la connessione nel file SLXSDataSyncServer.exe.config sia definita correttamente. Per saperne di più...
    • Le informazioni di connessione dovrebbero corrispondere a quelle di connection.config rilasciate per il proprio sito Web.
      Per impostazione predefinita il file di configurazione è in ...\Program Files\Saleslogix\SLXSDataSyncServer.exe.config e il file di connessione è rilasciato in ...\inetpub\wwwroot\slxclient
  • Verificare che la Configurazione della sincronizzazione SData è impostata correttamente con il giusto feed Infor CRM, nome utente e password.
  • Verificare che l'integrazione o le integrazioni sono impostate correttamente sulla vista Dettaglio Integrazione.
    • Verificare che la sincronizzazione è abilitata.
  • Aprire SLXSDataSyncServer.exe nella finestra del prompt dei comandi per visualizzare l'output dell'applicazione.
  • Visualizzare i registri degli errori, degli avvisi o i messagi di debug. Per saperne di più...
    • Nella Vista Dettaglio Sync Job aprire la scheda Elementi Log e filtrare per Tipo di Log per visualizzare errori, avvisi o messaggi di debug.
  • Sul Server Web esaminare il Visualizzatore Eventi di Windows per controllare se ci sono errori.
  • Abilitare il logging aggiuntivo sul Server Web. Per saperne di più...
    1. Nella cartella di installazione Infor CRM creare o aprire il file log4net.config per consentire la visualizzazione dei messaggi di debug di SLXSDataSyncServer nel Visualizzatore Eventi.
    2. Aggiungere o aggiornare le seguenti righe:
      <log4net threshold=''ALL''>
      <appender name=''EventLogAppender''
      type=''log4net.Appender.EventLogAppender'' >
      <applicationName value=''Services'' />
      <layout type=''log4net.Layout.PatternLayout''>
      <conversionPattern value=''%date [%thread] %-5level %logger
      [%property{NDC}] - %message%newline'' />
      </layout>
      </appender>

      <logger name=''SDataSync''>
      <level value=''ALL'' />
      <appender-ref ref=''EventLogAppender''/>
      </logger>
      </log4net>
    3. Nella cartella ...\Inetpub\wwwroot\SLXClient aprire il file log4net.config e ripetere il passaggio b. In tal modo sarà abilitata la registrazione dei messaggi di debug e di informazioni relativi ai processi SData avviati dal client. Ad esempio si riceve una chiamata dal Servizio Pricing.
    4. Nella cartella ...\Inetpub\wwwroot\sdata aprire il file log4net.config e cambiare il valore di livello di SDataSync da INFO a ALL oppure DEBUG in base ai messaggi che si desidera visualizzare nella schedaElementi Log.
      <logger name=''SDataSync''>
      <level value=''INFO'' />
      <appender-ref ref=''EventLogAppender''/>
