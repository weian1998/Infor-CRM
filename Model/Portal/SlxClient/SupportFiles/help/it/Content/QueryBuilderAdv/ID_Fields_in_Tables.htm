Campi ID nelle Tabelle

Ogni tabella ha almeno un campo ID, usato per assicurare che ogni record della tabella sia unico. Quando un record viene aggiunto alla tabella Azienda (ACCOUNT), un valore viene inserito automaticamente nel campo AccountID. Ogni volta in cui viene aggiunto un record viene generato un nuovo valore, basato sulle informazioni nel file chiave.

Quali siano i campi id di una tabella dipende dal tipo di relazione che la tabella ha con le altre. Ogni tabella creata che non è collegata con altre è dotata di un proprio campo ID. Se una nuova tabella viene creata come tabella secondaria con una relazione Uno a Uno con un'altra tabella, il suo campo ID sarà il campo ID della tabella principale. Se la tabella secondaria è in relazione Uno a Molti con un'altra tabella, occorre più di un campo per assicurare l'unicità. Perciò, durante la creazione della tabella verranno generati automaticamente due ID: un ID unico per la tabella ed il campo ID riportato dalla tabella principale. Se una tabella è una tabella di collegamento destinato a risolvere una relazione molti a molti, avrà bisogno di tre campi ID.Infor CRM solo generare automaticamente due ID, una unica per la tabella di collegamento e una per la tabella primaria. Occorre aggiungere il campo ID della tabella secondaria durante la creazione della tabella, oppure essa non collegherà la tabella principale a quella secondaria.

Nonostante alcune tabelle di collegamento esistenti in Infor CRM contengano soltanto due campi ID, sarebbe meglio includere tutti i tre campi ID nelle tabelle collegamento di propria creazione.
